Map Generation

Control how the server builds hex grids — biome distribution, resource scattering, terrain clusters, rivers, and spawn zones.

How Map Generation Works

When a game starts, the server generates the hex grid in a fixed pipeline:

  1. Allocate cells — create an empty hex grid of the configured size.
  2. Apply biome weights — assign each cell a biome based on the weight distribution you define.
  3. Generate clusters — override cells to create grouped patches of specific biomes.
  4. Carve rivers — lay down connected water-biome paths across the map.
  5. Scatter resources — randomly place resource deposits on eligible cells.
  6. Place neutral buildings — add capturable structures at valid locations.
  7. Clear spawn zones — remove obstacles near each player's starting position.

Each step is controlled by a dedicated DSL function inside the s.mapgen() block. The order you call them does not matter — the server always executes the pipeline in the order above.

s.mapgen(g => {
  g.size(48);

  // Biome weights (must sum to 1.0)
  g.biome("plains",   0.40);
  g.biome("forest",   0.25);
  g.biome("mountain", 0.20);
  g.biome("water",    0.10);
  g.biome("lava",     0.05);

  // Terrain clusters
  g.clusters("forest",   { minSize: 4, maxSize: 8, count: 6 });
  g.clusters("mountain", { minSize: 3, maxSize: 6, count: 4 });

  // Rivers
  g.rivers({ count: 2, minLength: 8, maxLength: 16 });

  // Resources
  g.scatter("gold", { biome: "mountain", density: 0.08 });
  g.scatter("wood", { biome: "forest",   density: 0.12 });

  // Neutral buildings
  g.placeNeutralBuilding("watchtower", { count: 4 });

  // Clear around spawn
  g.clearAroundStart(3);
});

g.biome(name, weight)

Define the base probability that any given cell will be assigned a specific biome during the initial distribution pass.

g.biome("plains",   0.40);
g.biome("forest",   0.25);
g.biome("mountain", 0.20);
g.biome("water",    0.10);
g.biome("lava",     0.05);
// Total: 1.00

Weights must sum to 1.0 (the server tolerates ±0.01 for floating-point rounding). If the total is outside this range, the server will reject the scenario at load time.

Weight Calculation Example

On a 48-hex grid with 2,304 total cells:

Biome Weight Expected cells
plains 0.40 ~922
forest 0.25 ~576
mountain 0.20 ~461
water 0.10 ~230
lava 0.05 ~115

These are expected counts before clusters and rivers modify the map. Actual numbers will vary per seed.

g.scatter(resource, { biome, density })

Randomly place resource deposits across cells of a given biome. Each eligible cell has a density probability of receiving a deposit.

g.scatter("gold", { biome: "mountain", density: 0.08 });
g.scatter("wood", { biome: "forest",   density: 0.12 });
Parameter Type Description
resource string Name of the resource to place (must match a defined s.resource()).
biome string Only place deposits on cells of this biome.
density number Probability per eligible cell, from 0.0 to 1.0. Typical range: 0.01–0.15.

g.clusters(biome, { minSize, maxSize, count })

Generate grouped patches of a biome, overriding whatever biome was originally assigned. Clusters create natural-looking terrain features like dense forests or mountain ranges.

g.clusters("forest",   { minSize: 4, maxSize: 8, count: 6 });
g.clusters("mountain", { minSize: 3, maxSize: 6, count: 4 });
Parameter Type Description
biome string The biome type for the cluster.
minSize number Minimum number of cells per cluster.
maxSize number Maximum number of cells per cluster.
count number How many clusters of this biome to generate.

g.rivers({ count, minLength, maxLength })

Carve river paths across the map. Rivers convert cells along their path to the water biome, creating natural barriers and strategic chokepoints.

g.rivers({ count: 2, minLength: 8, maxLength: 16 });
Parameter Type Description
count number Number of rivers to generate.
minLength number Minimum river length in cells.
maxLength number Maximum river length in cells.

g.clearAroundStart(radius)

Remove obstacles, hostile terrain, and neutral units within radius hexes of each player's spawn point. This ensures every player starts on navigable ground with room to build.

g.clearAroundStart(3);

The recommended radius is 2–4. A radius of 2 gives a tight start; 4 gives a comfortable buffer. Values below 2 risk spawning players on impassable terrain.

clearAroundStart is critical for fairness. Without it, some players may spawn surrounded by mountains or water with no room to expand, while others land on open plains. Always include this in competitive scenarios.

g.placeNeutralBuilding(type, { count, biome? })

Place capturable neutral buildings on the map. These give players strategic objectives to contest.

g.placeNeutralBuilding("watchtower", { count: 4 });
g.placeNeutralBuilding("gold_mine",  { count: 3, biome: "mountain" });
Parameter Type Description
type string Building type name (must match a defined s.building()).
count number How many to place on the map.
biome string Optional. Restrict placement to cells of this biome.

Seed for Reproducibility

The map generation seed is set in the server configuration (config.json), not in the scenario DSL. This allows the same scenario to produce different maps per game, or identical maps for testing.

// config.json
{
  "scenario": "castle",
  "mapSeed": 42
}

When mapSeed is set, every game with that scenario will produce the exact same map. Omit it for random generation each game.

Balance Tips

  • Always use clearAroundStart. This is the single most important fairness mechanism. Without it, spawn placement is a lottery.
  • Keep dominant biomes under 50%. A single biome above 50% weight makes the map feel monotonous. Aim for the primary biome at 30–40%.
  • Use clusters to create strategic terrain. Dense forest clusters become ambush zones; mountain ranges become natural walls. Clusters add tactical depth beyond raw biome weights.
  • Scale resources to map size. A density of 0.08 on a small map yields few deposits. Adjust density up for smaller maps or down for larger ones.
  • Rivers create natural borders. Two or three rivers can divide the map into contested sectors, encouraging conflict at crossing points.
  • Test with a fixed seed. During development, set mapSeed in your config so you can iterate on balance without map variance muddying the results.

Castle: Age of Kingdoms Example

The Castle scenario uses a 40/25/20/10/5 biome split with forest and mountain clusters and 2 rivers.

s.mapgen(g => {
  g.size(48);

  // Biome weights: 40/25/20/10/5
  g.biome("plains",   0.40);
  g.biome("forest",   0.25);
  g.biome("mountain", 0.20);
  g.biome("water",    0.10);
  g.biome("lava",     0.05);

  // Dense forest groves and mountain ranges
  g.clusters("forest",   { minSize: 4, maxSize: 8, count: 6 });
  g.clusters("mountain", { minSize: 3, maxSize: 6, count: 4 });

  // Two rivers winding across the map
  g.rivers({ count: 2, minLength: 8, maxLength: 16 });

  // Gold in mountains, wood in forests
  g.scatter("gold", { biome: "mountain", density: 0.08 });
  g.scatter("wood", { biome: "forest",   density: 0.12 });

  // Neutral watchtowers to contest
  g.placeNeutralBuilding("watchtower", { count: 4 });
  g.placeNeutralBuilding("gold_mine",  { count: 3, biome: "mountain" });

  // Fair starts
  g.clearAroundStart(3);
});

Cyberpunk: Scorched Network Example

The Cyberpunk scenario uses a 35/30/15/10/10 biome split with ruins and toxic clusters, no rivers, and neutral recycler buildings.

s.mapgen(g => {
  g.size(48);

  // Biome weights: 35/30/15/10/10
  g.biome("wasteland",   0.35);
  g.biome("ruins",       0.30);
  g.biome("toxic_zone",  0.15);
  g.biome("data_stream", 0.10);
  g.biome("bunker",      0.10);

  // Ruined city blocks and toxic waste dumps
  g.clusters("ruins",      { minSize: 5, maxSize: 10, count: 5 });
  g.clusters("toxic_zone", { minSize: 3, maxSize: 7,  count: 4 });

  // No rivers in the scorched wasteland

  // Scrap in ruins, energy in data streams
  g.scatter("scrap",  { biome: "ruins",       density: 0.10 });
  g.scatter("energy", { biome: "data_stream", density: 0.08 });

  // Neutral recyclers for resource conversion
  g.placeNeutralBuilding("recycler", { count: 5, biome: "ruins" });

  // Fair starts
  g.clearAroundStart(3);
});
