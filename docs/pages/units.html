Units

Creating and balancing units — from basic infantry to legendary bosses.

Full Parameter Reference

Pass these options as the second argument to s.unit().

Parameter Type Default Description
hp number — Hit points. The unit dies when HP reaches 0. Required.
attack number — Base attack damage dealt per strike. Required.
defense number 0 Flat damage reduction applied to incoming attacks.
range number 1 Attack range in hexes. 1 = melee, 2+ = ranged.
speed number 2 Movement points per turn. Spent against biome moveCost.
visionRange number 3 Number of hexes the unit can see (before biome modifiers).
cost object {} Resource cost to produce. Example: { gold: 50, iron: 10 }.
productionTime number 1 Number of turns required to produce this unit from a building.
color string — Hex color used to render the unit on the map.
abilities array [] List of ability IDs this unit can use. Each must be defined via s.ability().
carryCap number 0 Maximum resources this unit can carry (for harvester/transport roles).
upkeep object {} Resources drained per turn while the unit is alive. Example: { mana: 5 }.
neutral boolean false If true, the unit belongs to no player. Appears on the map as an obstacle or boss.
spawnGroup string null Group ID for neutral units that spawn together in clusters.
aoeRadius number 0 Area-of-effect radius for attacks. 0 = single target, 1+ = splash.
immunities array [] List of damage types or effects this unit is immune to (e.g., ["fire", "poison"]).
levels array [] Array of level-up definitions. See Level System below.
expPerKill number 0 Experience points awarded to the attacker when this unit is killed.

Unit Archetypes

Most units fall into one of six archetypes. Each archetype fills a distinct role on the battlefield. Mixing archetypes forces players to build diverse armies rather than spamming a single strong unit.

Melee

Close-range fighters with moderate HP and attack. They form the backbone of any army — cheap, fast to produce, and effective in numbers.

Ranged

Units with range: 2+ that deal damage from a distance. They are fragile up close, so they need melee units to screen for them.

Tank

High HP and defense, low speed. Tanks absorb damage and hold positions. They are expensive but hard to kill without focused fire or abilities.

Support

Units that buff allies or debuff enemies through abilities rather than raw stats. Often have upkeep costs to offset their utility.

Siege

Slow, powerful units with aoeRadius that excel at destroying buildings and clustered groups. Vulnerable to fast flankers.

Scout

Fast units with high speed and visionRange. Low combat stats, but invaluable for map control and early exploration.

Level System

The level system lets units grow stronger through combat. When the scenario's experienceSystem is enabled, units accumulate XP by killing enemies and level up when they hit the required thresholds.

Define levels as an array on the unit. Each entry specifies the XP required and the stat bonuses granted at that level:

s.unit("knight", {
  hp: 80,
  attack: 18,
  defense: 12,
  speed: 2,
  cost: { gold: 60 },
  color: "#c0c0c0",
  levels: [
    { expRequired: 20,  bonuses: { hp: 10, attack: 3 } },
    { expRequired: 50,  bonuses: { hp: 15, attack: 5, defense: 3 } },
    { expRequired: 100, bonuses: { hp: 20, attack: 8, defense: 5 } }
  ]
});

Bonuses are cumulative — a level-3 knight in the example above would have +45 HP, +16 attack, and +8 defense on top of its base stats.

Scale expRequired exponentially.

Linear XP thresholds let units snowball too fast. A good rule of thumb is to roughly double the requirement for each level: 20, 50, 100 is more balanced than 20, 40, 60.

Experience & XP Flow

Each unit can define expPerKill — the amount of experience the attacker earns when this unit dies. This creates a natural XP economy:

  • Weak units (skeletons, drones) give low XP — they are easy to farm but not very rewarding.
  • Strong units (knights, combat robots) give moderate XP — killing them is meaningful.
  • Boss units (dragons, rogue AIs) give high XP — defeating them is a major power spike.
// Low XP fodder
s.unit("skeleton", { hp: 20, attack: 5, expPerKill: 5, neutral: true });

// Medium XP standard unit
s.unit("knight", { hp: 80, attack: 18, expPerKill: 15 });

// High XP boss
s.unit("dragon", { hp: 300, attack: 40, expPerKill: 50, neutral: true });
Tie XP rewards to unit strength, not cost.

A unit that is hard to kill should give more XP regardless of its resource cost. This rewards skillful engagements over simply targeting cheap units.

Abilities on Units

The abilities array links a unit to one or more abilities defined with s.ability(). This is how you give units special moves, passive effects, and cooldown-based powers.

// First define the ability
s.ability("charge", {
  type: "active",
  damage: 25,
  range: 3,
  cooldown: 2,
  description: "Rush forward and strike with extra force."
});

// Then assign it to a unit
s.unit("knight", {
  hp: 80,
  attack: 18,
  defense: 12,
  speed: 2,
  cost: { gold: 60 },
  color: "#c0c0c0",
  abilities: ["charge"]
});

A unit can have multiple abilities. See the Abilities guide for the full ability parameter reference.

Neutral Units

Setting neutral: true makes a unit belong to no player. Neutral units appear on the map during generation and serve as obstacles, objectives, or environmental threats.

  • Neutral units do not require a cost — they are placed by the map generator, not produced by buildings.
  • Use spawnGroup to cluster neutral units together. All units with the same spawnGroup ID appear near each other on the map.
  • Neutral units can have abilities, XP rewards, and full combat stats just like player units.
s.unit("skeleton", {
  hp: 20,
  attack: 5,
  defense: 2,
  speed: 2,
  color: "#d4c9a8",
  neutral: true,
  spawnGroup: "undead_camp",
  expPerKill: 5
});

s.unit("dragon", {
  hp: 300,
  attack: 40,
  defense: 25,
  speed: 3,
  visionRange: 5,
  color: "#ff4444",
  neutral: true,
  immunities: ["fire"],
  abilities: ["flame_breath"],
  expPerKill: 50
});

Upkeep System

The upkeep parameter drains resources from the owning player every turn. This prevents unlimited army scaling and forces players to balance their economy against their military.

s.unit("mage", {
  hp: 40,
  attack: 22,
  defense: 4,
  range: 2,
  speed: 2,
  cost: { gold: 80 },
  upkeep: { mana: 5 },
  color: "#9c27b0",
  abilities: ["fireball", "shield"]
});
Use upkeep to gate powerful units.

A mage with mana upkeep stays balanced because the player cannot field an unlimited number of them — eventually the mana economy runs dry. This creates a natural soft cap on elite units.

Balance Tips

Use this table as a starting point for each archetype. Adjust based on your scenario's resource economy and pacing.

Archetype HP Attack Defense Range Speed Cost (Gold)
Melee 50–80 10–20 5–15 1 2–3 20–60
Ranged 30–50 12–22 2–8 2–3 2 30–70
Tank 100–200 8–15 15–30 1 1–2 60–120
Support 30–50 5–12 3–10 1–2 2 40–80
Siege 60–100 25–40 5–10 2–3 1 80–150
Scout 20–40 5–10 2–5 1 4–5 15–40

Castle Scenario Examples

The Castle scenario features a classic fantasy army with six unit types.

Knight (Melee Tank)

The backbone of the Castle army. High HP, solid defense, and the charge ability for burst damage.

s.unit("knight", {
  hp: 80,
  attack: 18,
  defense: 12,
  range: 1,
  speed: 2,
  cost: { gold: 60 },
  color: "#c0c0c0",
  abilities: ["charge"],
  expPerKill: 15,
  levels: [
    { expRequired: 20,  bonuses: { hp: 10, attack: 3 } },
    { expRequired: 50,  bonuses: { hp: 15, attack: 5, defense: 3 } },
    { expRequired: 100, bonuses: { hp: 20, attack: 8, defense: 5 } }
  ]
});

Archer (Ranged)

Long-range damage dealer. Fragile in melee but punishes units that advance without cover.

s.unit("archer", {
  hp: 35,
  attack: 14,
  defense: 3,
  range: 3,
  speed: 2,
  visionRange: 4,
  cost: { gold: 40 },
  color: "#8bc34a",
  expPerKill: 10,
  levels: [
    { expRequired: 15, bonuses: { attack: 3, range: 1 } },
    { expRequired: 40, bonuses: { attack: 5, hp: 10 } }
  ]
});

Mage (Caster with Upkeep)

Powerful ranged attacker with abilities, gated by mana upkeep. The glass cannon of the roster.

s.unit("mage", {
  hp: 40,
  attack: 22,
  defense: 4,
  range: 2,
  speed: 2,
  cost: { gold: 80 },
  upkeep: { mana: 5 },
  color: "#9c27b0",
  abilities: ["fireball", "shield"],
  expPerKill: 18,
  levels: [
    { expRequired: 25, bonuses: { attack: 5 } },
    { expRequired: 60, bonuses: { attack: 8, hp: 10 } }
  ]
});

Catapult (Siege AOE)

Slow but devastating. Area-of-effect attacks smash buildings and clustered units alike.

s.unit("catapult", {
  hp: 70,
  attack: 30,
  defense: 5,
  range: 3,
  speed: 1,
  aoeRadius: 1,
  cost: { gold: 120, wood: 40 },
  productionTime: 3,
  color: "#795548",
  expPerKill: 20
});

Dragon (Neutral Boss)

A fearsome neutral boss guarding high-value map locations. Immune to fire, equipped with flame breath, and worth massive XP.

s.unit("dragon", {
  hp: 300,
  attack: 40,
  defense: 25,
  range: 2,
  speed: 3,
  visionRange: 5,
  color: "#ff4444",
  neutral: true,
  immunities: ["fire"],
  abilities: ["flame_breath"],
  expPerKill: 50
});

Skeleton (Neutral Swarm)

Weak neutral units that spawn in groups. Easy XP for early-game leveling, but dangerous in numbers if ignored.

s.unit("skeleton", {
  hp: 20,
  attack: 5,
  defense: 2,
  range: 1,
  speed: 2,
  color: "#d4c9a8",
  neutral: true,
  spawnGroup: "undead_camp",
  expPerKill: 5
});

Cyberpunk Scenario Examples

The Cyberpunk scenario replaces swords and sorcery with drones, hackers, and rogue AIs.

Drone (Fast Scout)

Cheap, fast, and disposable. Excellent for early scouting and map control.

s.unit("drone", {
  hp: 25,
  attack: 8,
  defense: 2,
  range: 1,
  speed: 5,
  visionRange: 5,
  cost: { credits: 15 },
  color: "#00bcd4",
  expPerKill: 4
});

Combat Robot (Versatile)

A well-rounded mid-tier unit. Solid in both offense and defense with no major weaknesses.

s.unit("combat_robot", {
  hp: 70,
  attack: 16,
  defense: 10,
  range: 2,
  speed: 2,
  cost: { credits: 55, alloy: 15 },
  color: "#607d8b",
  expPerKill: 14,
  levels: [
    { expRequired: 20, bonuses: { attack: 4, defense: 2 } },
    { expRequired: 50, bonuses: { hp: 20, attack: 6 } }
  ]
});

Hacker (Building Attacker)

Specialized unit that excels at disabling and destroying enemy buildings. Weak in direct combat but devastating against infrastructure.

s.unit("hacker", {
  hp: 30,
  attack: 10,
  defense: 3,
  range: 2,
  speed: 2,
  cost: { credits: 50 },
  upkeep: { bandwidth: 3 },
  color: "#76ff03",
  abilities: ["system_hack", "disable_turret"],
  expPerKill: 12
});

Colossus (Heavy)

The Cyberpunk tank. Massive HP and area damage make it a slow-moving wrecking ball.

s.unit("colossus", {
  hp: 200,
  attack: 28,
  defense: 20,
  range: 1,
  speed: 1,
  aoeRadius: 1,
  cost: { credits: 140, alloy: 50 },
  productionTime: 4,
  color: "#455a64",
  expPerKill: 25,
  levels: [
    { expRequired: 30, bonuses: { hp: 30, defense: 5 } },
    { expRequired: 70, bonuses: { hp: 40, attack: 10 } }
  ]
});

Rogue AI (Neutral)

A dangerous neutral boss that controls a zone of the map. High XP reward for the team that takes it down.

s.unit("rogue_ai", {
  hp: 250,
  attack: 35,
  defense: 20,
  range: 3,
  speed: 2,
  visionRange: 6,
  color: "#ff1744",
  neutral: true,
  abilities: ["emp_blast", "self_repair"],
  expPerKill: 45
});

Mutant (Neutral, Toxic)

Neutral units immune to poison that roam toxic zones. They spawn in packs and are resistant to environmental damage.

s.unit("mutant", {
  hp: 45,
  attack: 12,
  defense: 6,
  range: 1,
  speed: 3,
  color: "#aeea00",
  neutral: true,
  spawnGroup: "mutant_pack",
  immunities: ["poison"],
  expPerKill: 8
});
