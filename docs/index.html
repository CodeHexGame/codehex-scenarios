CodeHex Scenarios

Create custom game content for CodeHex using a simple declarative DSL. No server-side code — just describe your world.

What is a Scenario?

A scenario is a single scenario.js file that declares all game content: biomes, resources, units, buildings, abilities, events, map generation, graphics, and audio. The server loads it into a sandboxed environment — no require, no fs, no process.

scenario("my_scenario", s => {
  s.name("My Scenario");
  s.biome("plains", { color: "#8db360", moveCost: 1 });
  s.resource("gold", { color: "#ffd700", rarity: "common" });
  s.unit("warrior", { hp: 60, attack: 12, defense: 8, cost: { gold: 30 }, color: "#f44336" });
  // ... more declarations
});

Get Started in 3 Steps

Copy the template

cp -r template/ my_scenario/

The template includes every DSL function with full comments and example values. It runs as-is.

Customize your scenario

Open my_scenario/scenario.js and change the biomes, units, buildings — everything. See the DSL Reference for all available functions.

Deploy to server

cp -r my_scenario/ codehex-server/scenarios/my_scenario/

Set "scenario": "my_scenario" in config.json and start the server.

For a more detailed walkthrough, see the Quick Start guide.

Official Scenarios

Castle: Age of Kingdoms

Medieval fantasy with knights, archers, mages, and dragons. Features morale, experience, day/night cycles, and weather.

Victory: Domination (70% map control)

Biomes: plains, forest, mountain, water, lava

Units: knight, archer, mage, catapult, dragon, skeleton

Cyberpunk: Scorched Network

Post-apocalyptic sci-fi with drones, combat robots, hackers, and rogue AI. Features logistics mode, siege warfare, and radiation.

Victory: Score (1000 points)

Biomes: wasteland, ruins, toxic_zone, data_stream, bunker

Units: drone, combat_robot, hacker, colossus, rogue_ai, mutant

See the Examples page for full annotated walkthroughs of both scenarios.
