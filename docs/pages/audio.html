Audio

Add background music and sound effects to make your scenario immersive.

Format

All audio files must be in OGG Vorbis format (.ogg). OGG provides the best cross-browser compatibility for web-based game audio. MP3 and WAV are not supported by the engine.

Most audio editors (Audacity, etc.) can export to OGG. Use a bitrate of 128-192 kbps for music and 96 kbps for short sound effects to keep file sizes reasonable.

Background Music

Background music is defined as an array of OGG files. The engine plays them in sequence and loops back to the first track when the list is exhausted, creating a continuous music cycle:

s.audio({
  music: [
    "theme_intro.ogg",
    "theme_battle.ogg",
    "theme_exploration.ogg"
  ]
});

The tracks crossfade automatically. A single-track array will loop that track indefinitely.

Sound Effects

Sound effects are named sounds triggered by in-game events. Each name maps to an OGG file:

s.audio({
  sfx: {
    attack:   "sfx_attack.ogg",
    build:    "sfx_build.ogg",
    death:    "sfx_death.ogg",
    levelup:  "sfx_levelup.ogg",
    event:    "sfx_event.ogg",
    produce:  "sfx_produce.ogg",
    select:   "sfx_select.ogg"
  }
});

Standard Sound Names

The engine recognizes these sound names and triggers them automatically at the appropriate moments:

NameTriggered When
attackA unit performs an attack action.
buildA building is placed or construction completes.
deathA unit or building is destroyed.
levelupA unit gains a level from experience.
eventA random event is triggered on the map.
produceA building finishes producing a unit.
selectThe player selects a unit or building.

Custom Sound Names

You can define any additional sound name as a string key. Custom sounds are triggered via events or abilities using the playSound action:

s.audio({
  sfx: {
    // Standard sounds
    attack: "sfx_attack.ogg",
    death:  "sfx_death.ogg",
    // Custom sounds
    dragon_roar:   "sfx_dragon_roar.ogg",
    spell_cast:    "sfx_spell_cast.ogg",
    treasure_open: "sfx_treasure.ogg"
  }
});

// Trigger a custom sound from an event
s.event("dragon_appears", {
  chance: 0.05,
  effect: { playSound: "dragon_roar" }
});

Volume and Playback

Volume levels and playback controls (mute, pause, master volume) are managed entirely by the client application, not by the scenario file. Your scenario only declares which sounds exist and when they should play. The player adjusts volume through in-game settings.

Keep music tracks at a consistent loudness level across your scenario. The client applies a single master volume ‚Äî it cannot adjust individual tracks independently.

Where to Get Free Sounds

freesound.org

Massive community library of sound effects. Most assets are CC0 or CC-BY. Use the search filters to find game-ready clips.

kenney.nl

Audio packs alongside the sprite packs ‚Äî UI sounds, impacts, ambient loops. All CC0 (public domain).

opengameart.org

Music tracks and sound effects specifically made for games. Check the license per asset ‚Äî CC0 and CC-BY are common.

Examples

Castle: Age of Kingdoms

The Castle scenario uses orchestral music with medieval-themed sound effects:

s.audio({
  music: [
    "castle_theme.ogg",
    "castle_battle.ogg",
    "castle_peace.ogg"
  ],
  sfx: {
    attack:   "sword_clash.ogg",
    build:    "hammer_build.ogg",
    death:    "death_cry.ogg",
    levelup:  "fanfare.ogg",
    event:    "horn_blast.ogg",
    produce:  "anvil_produce.ogg",
    select:   "armor_clink.ogg",
    dragon_roar: "dragon_roar.ogg"
  }
});

Cyberpunk: Scorched Network

The Cyberpunk scenario uses electronic music with sci-fi sound effects:

s.audio({
  music: [
    "cyber_ambient.ogg",
    "cyber_combat.ogg",
    "cyber_tension.ogg"
  ],
  sfx: {
    attack:   "laser_shot.ogg",
    build:    "construct_beep.ogg",
    death:    "explosion_small.ogg",
    levelup:  "upgrade_chime.ogg",
    event:    "alert_siren.ogg",
    produce:  "assembly_whir.ogg",
    select:   "ui_click.ogg",
    hack_complete: "glitch_resolve.ogg",
    radiation_warning: "geiger_tick.ogg"
  }
});

Audio is entirely optional. Your scenario works perfectly without any sound files ‚Äî the engine simply plays nothing when no audio is configured.
