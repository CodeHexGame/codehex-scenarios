DSL Reference

Complete reference for every function available inside scenario("id", s => { ... }). All functions are called on the scenario builder object s.

Tip Use the search box above to filter functions by name, parameter, or keyword. The table of contents on the right links to each entry.

s.ability()

s.ability(name: string, options: object)
Defines a special ability that can be assigned to units. Abilities have cooldowns, costs, and can deal damage, heal, or apply status effects in an area.
ParameterTypeDefaultDescription
namestringrequiredUnique identifier for this ability.
cooldownnumber1Number of turns before the ability can be used again.
rangenumber1How many hexes away the ability can reach.
damagenumber0Base damage dealt to the target.
damageMultinumber1.0Multiplier applied to the base damage.
aoeRadiusnumber0Radius of the area-of-effect in hexes. 0 means single-target.
costobject{}Resource cost to activate, e.g. { mana: 20 }.
damageTypestring"physical"Type of damage: "physical", "magic", "fire", "poison", etc.
effectstringnullName of a status effect to apply on hit (must match a defined s.statusEffect).
effectDurationnumber1Number of turns the applied status effect lasts.
targetsCountnumber1Maximum number of targets the ability can hit (within AOE).
targetTypestring"enemy"Who can be targeted: "enemy", "friendly", "all", "self".
healAmountnumber0HP restored to the target. Used for healing abilities.
 
s.ability("fireball", {
  cooldown: 3,
  range: 4,
  damage: 25,
  damageType: "fire",
  aoeRadius: 2,
  cost: { mana: 30 },
  effect: "burning",
  effectDuration: 2,
  targetsCount: 5,
  targetType: "enemy"
});

s.audio()

s.audio(config: object)
Configures the scenario's music playlist and sound effects. Music tracks loop in order; sounds are triggered by game events.
ParameterTypeDefaultDescription
musicstring[][]Array of music file paths relative to the scenario's audio/ folder.
soundsobject{}Map of event names to sound file paths, e.g. { attack: "sfx/hit.ogg" }.
 
s.audio({
  music: [
    "audio/theme_main.ogg",
    "audio/theme_battle.ogg"
  ],
  sounds: {
    attack: "audio/sfx/sword.ogg",
    build: "audio/sfx/hammer.ogg",
    death: "audio/sfx/death.ogg",
    victory: "audio/sfx/fanfare.ogg"
  }
});

s.aura()

s.aura(name: string, options: object)
Defines a named aura that can be emitted by buildings or units. Auras apply passive effects to all friendly units within a radius each turn.
ParameterTypeDefaultDescription
namestringrequiredUnique identifier for this aura.
effectobjectrequiredStat modifiers applied to affected units, e.g. { attack: 3, defense: 2 }.
stackablebooleanfalseWhether multiple instances of this aura stack their effects.
 
s.aura("rallying_cry", {
  effect: { attack: 5, defense: 2 },
  stackable: false
});

s.biome()

s.biome(name: string, options: object)
Defines a terrain type that tiles the hex map. Biomes affect movement, combat, vision, and resource harvesting. At least one biome must be defined for a valid scenario.
ParameterTypeDefaultDescription
namestringrequiredUnique identifier for this biome.
colorstringrequiredHex color used to render this biome on the map, e.g. "#8db360".
moveCostnumber1Movement points required to enter this tile. Higher values slow units down.
defenseBonusnumber0Flat bonus added to a defending unit's defense stat on this tile.
attackBonusnumber0Flat bonus added to an attacking unit's attack stat on this tile.
harvestBonusnumber0Extra resources gained when harvesting on this tile.
visionBonusnumber0Extra vision range granted to units standing on this tile.
visionPenaltynumber0Reduction in vision range when looking into this tile (e.g. dense forest).
damagePerTurnnumber0HP lost by any unit standing on this tile at the start of each turn.
impassablebooleanfalseIf true, no unit can enter this tile (e.g. deep water, void).
allowedUnitsstring[]nullIf set, only the listed unit types may enter. null means all units allowed.
spawnWeightnumber1Relative probability weight used by the map generator when placing this biome.
 
s.biome("forest", {
  color: "#2d5a27",
  moveCost: 2,
  defenseBonus: 3,
  visionPenalty: 1,
  harvestBonus: 1,
  spawnWeight: 3
});

s.biome("lava", {
  color: "#ff4500",
  moveCost: 3,
  damagePerTurn: 10,
  impassable: false,
  spawnWeight: 0.5
});

s.building()

s.building(name: string, options: object)
Defines a building that players can construct. Buildings can produce units, provide auras, store resources, auto-attack enemies, and be upgraded through levels.
ParameterTypeDefaultDescription
namestringrequiredUnique identifier for this building.
hpnumber100Maximum hit points of the building.
defensenumber5Flat damage reduction applied to incoming attacks.
sizenumber1Number of hex tiles this building occupies.
costobjectrequiredResource cost to construct, e.g. { gold: 100, wood: 50 }.
producesstring[][]List of unit names this building can train.
visionRangenumber3How far the building reveals the fog of war.
aurastringnullName of an aura this building emits (must match a defined s.aura).
auraRadiusnumber2Radius in hexes for the building's aura.
auraEffectobjectnullInline aura effect object (alternative to referencing a named aura).
storageobject{}Resource storage capacity, e.g. { gold: 500 }. Increases the player's resource cap.
maxLevelnumber1Maximum upgrade level for this building.
levelsobject[][]Array of per-level stat overrides, e.g. [{ hp: 150, defense: 8 }].
passiveHarvestobjectnullResources generated each turn, e.g. { gold: 5 }.
mustBePlacedOnstring[]nullRestrict placement to specific biomes, e.g. ["plains", "forest"].
passablebooleanfalseIf true, any unit can walk through this building's tile.
ownTeamPassablebooleantrueIf true, the owning player's units can walk through this building.
autoAttackbooleanfalseIf true, the building automatically attacks nearby enemies each turn.
attacknumber0Base attack damage when autoAttack is enabled.
rangenumber2Attack range in hexes when autoAttack is enabled.
productionSpeedBonusnumber0Flat reduction in production time for all units trained here.
 
s.building("barracks", {
  hp: 200,
  defense: 10,
  cost: { gold: 120, wood: 60 },
  produces: ["warrior", "archer", "knight"],
  visionRange: 4,
  storage: { gold: 200 },
  maxLevel: 3,
  levels: [
    { hp: 250, defense: 12, productionSpeedBonus: 1 },
    { hp: 300, defense: 15, productionSpeedBonus: 2 }
  ],
  mustBePlacedOn: ["plains", "forest"]
});

s.building("watchtower", {
  hp: 80,
  defense: 5,
  cost: { gold: 60, wood: 40 },
  visionRange: 6,
  autoAttack: true,
  attack: 8,
  range: 3
});

s.description()

s.description(text: string)
Sets the scenario's description shown in the lobby and scenario browser. Supports plain text only. Keep it under 200 characters for best display.
ParameterTypeDefaultDescription
textstringrequiredThe scenario description text.
 
s.description("A medieval fantasy scenario with knights, castles, and dragons. Conquer the realm!");

s.event()

s.event(name: string, options: object)
Defines a random event that can trigger during gameplay. Events have a chance to fire each turn and can modify the game state through an effect callback. The callback receives an event context e with helper methods.
ParameterTypeDefaultDescription
namestringrequiredUnique identifier for this event.
chancenumber0.1Probability (0–1) that this event triggers each turn.
minStepnumber1Earliest turn number on which this event can trigger.
maxPerGamenumberInfinityMaximum number of times this event can fire in a single game.
messagestring""Text displayed to all players when the event fires.
effectfunctionrequiredCallback (e) => { ... } that modifies game state. See below for e.* methods.

Event context methods (e.*)

MethodDescription
e.damageAll(amount)Deal damage to every unit on the map.
e.healAll(amount)Heal every unit on the map.
e.giveAll(resource, amount)Give resources to every player.
e.takeAll(resource, amount)Remove resources from every player.
e.spawnNeutral(unitName, count)Spawn neutral units at random positions.
e.applyOverlay(overlayName, count)Place overlay tiles at random positions.
e.changeBiome(from, to, count)Convert random tiles from one biome to another.
e.buffAll(stat, amount, duration)Temporarily boost a stat for all units.
 
s.event("meteor_shower", {
  chance: 0.05,
  minStep: 10,
  maxPerGame: 3,
  message: "Meteors rain from the sky!",
  effect: e => {
    e.damageAll(15);
    e.changeBiome("plains", "lava", 5);
    e.spawnNeutral("fire_elemental", 2);
  }
});

s.mapgen()

s.mapgen(callback: function)
Configures procedural map generation. The callback receives a generator context g with methods to control biome placement, terrain features, and starting positions.
MethodSignatureDescription
g.biome(name, weight)Override the biome's spawn weight for map generation.
g.scatter(biome, density)Scatter individual tiles of a biome randomly. Density is 0–1.
g.clusters(biome, count, size)Place clustered groups of a biome. Size is average tiles per cluster.
g.rivers(biome, count)Generate meandering river lines using the given biome.
g.clearAroundStart(radius, biome)Replace tiles around each player's start position with the given biome.
g.placeNeutralBuilding(building, count)Place neutral (unowned) buildings at random valid positions.
 
s.mapgen(g => {
  g.biome("plains", 5);
  g.biome("forest", 3);
  g.biome("mountain", 2);
  g.scatter("water", 0.08);
  g.clusters("forest", 6, 8);
  g.rivers("water", 2);
  g.clearAroundStart(3, "plains");
  g.placeNeutralBuilding("gold_mine", 4);
});

ParameterTypeDefaultDescription

s.observers()

s.observers(options: object)
Configures observer (spectator) settings for the game. Controls whether observers are allowed, their visibility, and the maximum number.
ParameterTypeDefaultDescription
allowedbooleantrueWhether observers can join the game.
fogOfWarbooleanfalseIf true, observers see fog of war (same as players). If false, observers see the entire map.
maxCountnumber10Maximum number of simultaneous observers.
 
s.observers({
  allowed: true,
  fogOfWar: false,
  maxCount: 5
});

s.overlay()

s.overlay(name: string, options: object)
Defines an overlay that can be placed on top of biome tiles. Overlays are semi-transparent effects like fire, poison clouds, or radiation zones that can spread and deal damage.
ParameterTypeDefaultDescription
namestringrequiredUnique identifier for this overlay.
colorstringrequiredHex color used to render the overlay, e.g. "#ff440088".
opacitynumber0.5Opacity of the overlay (0–1).
damagePerTurnnumber0HP damage dealt to any unit standing on this overlay each turn.
spreadChancenumber0Probability (0–1) that this overlay spreads to an adjacent tile each turn.
durationnumberInfinityNumber of turns before the overlay disappears. Infinity means permanent.
effectstringnullName of a status effect applied to units on this overlay.
 
s.overlay("wildfire", {
  color: "#ff4400",
  opacity: 0.6,
  damagePerTurn: 8,
  spreadChance: 0.3,
  duration: 5,
  effect: "burning"
});

s.resource()

s.resource(name: string, options: object)
Defines a resource type that can be harvested, spent, and stored. Resources appear as deposits on the map and are the economic backbone of the game.
ParameterTypeDefaultDescription
namestringrequiredUnique identifier for this resource.
colorstringrequiredHex color used to render resource deposits on the map.
raritystring"common"Controls spawn frequency: "common", "uncommon", "rare", or "legendary".
maxDepositnumber100Maximum amount in a single resource deposit.
regenRatenumber0Amount regenerated per turn in each deposit. 0 means non-renewable.
harvestAmountnumber10Amount collected per harvest action by a unit.
decayRatenumber0Amount lost from a player's stockpile each turn (e.g. perishable goods).
 
s.resource("gold", {
  color: "#ffd700",
  rarity: "common",
  maxDeposit: 200,
  regenRate: 2,
  harvestAmount: 15
});

s.resource("crystal", {
  color: "#e040fb",
  rarity: "rare",
  maxDeposit: 50,
  harvestAmount: 5
});

s.resourceCap()

s.resourceCap(name: string, amount: number)
Sets the maximum amount of a resource a player can hold at once (before building storage). Excess resources are lost. Call once per resource.
ParameterTypeDefaultDescription
namestringrequiredName of the resource (must match a defined s.resource).
amountnumberrequiredMaximum storage capacity for this resource.
 
s.resourceCap("gold", 500);
s.resourceCap("wood", 300);
s.resourceCap("crystal", 100);

s.respawn()

s.respawn(options: object)
Configures player respawn behavior when a player is eliminated. Controls whether players can come back, with what resources, and after how long.
ParameterTypeDefaultDescription
allowedbooleanfalseWhether eliminated players can respawn.
delaynumber3Number of turns before the player respawns.
startFreshbooleantrueIf true, the player restarts with default start conditions. If false, they keep accumulated resources.
keepScorebooleantrueIf true, the player's score is preserved across respawns.
 
s.respawn({
  allowed: true,
  delay: 5,
  startFresh: true,
  keepScore: false
});

s.rules()

s.rules(options: object)
Sets global gameplay rule flags. These toggle major game mechanics on or off for the entire scenario.
ParameterTypeDefaultDescription
friendlyFirebooleanfalseAllow units to damage friendly units and buildings.
fogOfWarbooleantrueEnable fog of war. If false, the entire map is visible.
dayNightCyclebooleanfalseEnable day/night cycle, which affects vision ranges.
weatherbooleanfalseEnable random weather events that affect combat and movement.
moralebooleanfalseEnable the morale system. Units gain or lose morale based on combat outcomes.
experiencebooleanfalseEnable the XP and leveling system for units.
logisticsbooleanfalseEnable supply lines. Units far from buildings have reduced effectiveness.
siegeModebooleanfalseEnable siege mechanics. Some units can enter siege mode for bonus range/damage at the cost of mobility.
diplomacybooleanfalseEnable alliances and diplomacy between players.
 
s.rules({
  fogOfWar: true,
  dayNightCycle: true,
  weather: true,
  morale: true,
  experience: true,
  friendlyFire: false,
  logistics: false,
  siegeMode: false,
  diplomacy: false
});

s.sprites()

s.sprites(config: object)
Maps game entities to sprite image files. Each key in the config maps an entity type to an object of name-to-file-path pairs. Paths are relative to the scenario's sprites/ folder.
ParameterTypeDefaultDescription
unitsobject{}Map of unit names to sprite paths, e.g. { warrior: "units/warrior.png" }.
buildingsobject{}Map of building names to sprite paths.
biomesobject{}Map of biome names to tile sprite paths. Overrides the flat color.
resourcesobject{}Map of resource names to deposit sprite paths.
effectsobject{}Map of effect/ability names to animation sprite paths.
 
s.sprites({
  units: {
    warrior: "sprites/units/warrior.png",
    archer: "sprites/units/archer.png",
    mage: "sprites/units/mage.png"
  },
  buildings: {
    barracks: "sprites/buildings/barracks.png",
    tower: "sprites/buildings/tower.png"
  },
  biomes: {
    forest: "sprites/tiles/forest.png",
    mountain: "sprites/tiles/mountain.png"
  },
  resources: {
    gold: "sprites/resources/gold.png"
  },
  effects: {
    fireball: "sprites/fx/fireball.png"
  }
});

s.startCondition()

s.startCondition(callback: function)
Defines what each player starts with when the game begins (or when they respawn with startFresh: true). The callback receives a player context p with setup methods.
MethodSignatureDescription
p.give(resource, amount)Give the player starting resources.
p.spawnUnit(unitName, count)Spawn units near the player's starting position.
p.buildBuilding(buildingName)Place a pre-built building at the player's starting position.
p.nearPos(dx, dy)Set a relative offset from the start position for the next spawn/build call.
 
s.startCondition(p => {
  p.give("gold", 200);
  p.give("wood", 100);
  p.buildBuilding("town_hall");
  p.spawnUnit("warrior", 2);
  p.spawnUnit("worker", 3);
  p.nearPos(2, 0);
  p.spawnUnit("scout", 1);
});

s.statusEffect()

s.statusEffect(name: string, options: object)
Defines a status effect (buff or debuff) that can be applied to units by abilities, overlays, or events. Status effects persist for a set duration and modify unit behavior each turn.
ParameterTypeDefaultDescription
namestringrequiredUnique identifier for this status effect.
durationnumber3Default number of turns the effect lasts.
blocksMovebooleanfalseIf true, the affected unit cannot move.
blocksAttackbooleanfalseIf true, the affected unit cannot attack.
damagePerTurnnumber0HP damage dealt to the affected unit at the start of each turn.
damageTypestring"physical"Type of damage dealt by damagePerTurn.
colorstring"#ffffff"Color of the visual indicator shown on affected units.
 
s.statusEffect("burning", {
  duration: 3,
  damagePerTurn: 5,
  damageType: "fire",
  color: "#ff6600"
});

s.statusEffect("frozen", {
  duration: 2,
  blocksMove: true,
  blocksAttack: true,
  color: "#00ccff"
});

s.unit()

s.unit(name: string, options: object)
Defines a unit type that can be trained and controlled by players (or spawned as neutrals). Units are the primary agents on the map — they move, fight, harvest, and use abilities.
ParameterTypeDefaultDescription
namestringrequiredUnique identifier for this unit type.
hpnumber50Maximum hit points.
attacknumber10Base attack damage per hit.
defensenumber5Flat damage reduction from incoming attacks.
rangenumber1Attack range in hexes. 1 = melee, 2+ = ranged.
speednumber3Movement points per turn. Tiles cost moveCost points to enter.
visionRangenumber4How far the unit can see through fog of war, in hexes.
costobjectrequiredResource cost to train, e.g. { gold: 50 }.
productionTimenumber1Number of turns to train this unit at a building.
colorstring"#ffffff"Fallback color when no sprite is assigned.
abilitiesstring[][]List of ability names this unit can use (must match defined s.ability entries).
carryCapnumber0Maximum resources this unit can carry. 0 means it cannot harvest.
upkeepobject{}Resources consumed each turn to maintain this unit, e.g. { gold: 2 }. Unit is damaged if upkeep cannot be paid.
neutralbooleanfalseIf true, this unit type is hostile to all players (NPC / creep).
spawnGroupnumber1Number of units spawned together when a neutral spawn occurs.
aoeRadiusnumber0If > 0, basic attacks deal splash damage in this radius.
immunitiesstring[][]List of status effect names this unit is immune to.
levelsobject[][]Per-level stat overrides, e.g. [{ hp: 60, attack: 14 }, { hp: 75, attack: 18 }]. Requires rules.experience = true.
expPerKillnumber10Experience points awarded for killing this unit type.
 
s.unit("knight", {
  hp: 80,
  attack: 18,
  defense: 12,
  range: 1,
  speed: 3,
  visionRange: 4,
  cost: { gold: 80, iron: 20 },
  productionTime: 2,
  color: "#c0c0c0",
  abilities: ["charge"],
  carryCap: 0,
  immunities: ["fear"],
  levels: [
    { hp: 100, attack: 22, defense: 14 },
    { hp: 120, attack: 26, defense: 16 }
  ],
  expPerKill: 25
});

s.unit("worker", {
  hp: 30,
  attack: 3,
  defense: 2,
  speed: 4,
  cost: { gold: 20 },
  carryCap: 20,
  color: "#8b6914"
});

s.victoryCondition()

s.victoryCondition(type: string, options?: object)
Sets the win condition for the game. Only one victory condition can be active. The type determines which options are relevant.
ParameterTypeDefaultDescription
typestringrequiredVictory type: "domination", "score", "elimination", "capture", or "survival".
options.thresholdnumbervariesTarget value. For "domination": map control % (e.g. 0.7). For "score": point target. For "survival": turns to survive.
options.targetstringnullFor "capture": the building name that must be captured to win.
options.turnLimitnumberInfinityMaximum turns before the game ends. Highest score wins if no condition is met.
 
// Domination: control 70% of the map
s.victoryCondition("domination", { threshold: 0.7 });

// Score: reach 1000 points, max 100 turns
s.victoryCondition("score", { threshold: 1000, turnLimit: 100 });

// Elimination: last player standing
s.victoryCondition("elimination");

// Capture: seize the enemy's throne
s.victoryCondition("capture", { target: "throne" });

// Survival: survive 50 turns
s.victoryCondition("survival", { threshold: 50 });
