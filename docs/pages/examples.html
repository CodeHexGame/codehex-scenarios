Examples

Learning from the official scenarios — full annotated walkthroughs of both Castle and Cyberpunk, showing how the same DSL creates completely different gameplay experiences.

Castle: Age of Kingdoms

A classic medieval fantasy scenario featuring knights, archers, mages, and dragons. Players build castles, train armies, and conquer territory. The first player to control 70% of the map wins.

Meta and Rules Setup

The Castle scenario enables most of the advanced gameplay systems. Morale keeps armies fighting effectively, experience rewards veteran units, day/night cycles change visibility, weather adds randomness, and siege mode lets catapults destroy buildings.

scenario("castle", s => {

  s.name("Castle: Age of Kingdoms");
  s.description("Medieval fantasy with knights, dragons, and siege warfare.");
  s.version("1.0.0");
  s.author("CodeHex Team");

  s.rules({
    friendlyFire:       false,
    logisticsMode:      false,
    moraleSystem:       true,
    experienceSystem:   true,
    resourceCaps:       true,
    globalResourcePool: false,
    nightCycle:         true,
    weather:            true,
    siegeMode:          true,
  });
Why no logistics mode?

In a medieval setting, resources flow to the player pool automatically — there is no need for units to physically carry them back to base. This keeps the focus on combat and territory control rather than supply-line management.

Biomes

Five biomes create a balanced fantasy battlefield. Plains are easy to cross, forests reward defensive play, mountains offer strategic high ground, water shapes pathing, and lava creates a niche for the dragon.

  s.biome("plains", {
    color: "#8db360", moveCost: 1,
    defenseBonus: 0, attackBonus: 0,
    harvestBonus: 10, spawnWeight: 0.3,
  });

  s.biome("forest", {
    color: "#2d5a27", moveCost: 2,
    defenseBonus: 20, visionPenalty: 1, spawnWeight: 0.5,
  });

  s.biome("mountain", {
    color: "#8a8a8a", moveCost: 3,
    defenseBonus: 30, attackBonus: 10,
    visionBonus: 2, spawnWeight: 0.1,
  });

  s.biome("water", {
    color: "#3a7bd5", impassable: true,
  });

  s.biome("lava", {
    color: "#ff4500", moveCost: 2,
    damagePerTurn: 10, allowedUnits: ["dragon"], spawnWeight: 0.0,
  });
Balance choices

Forest has the highest spawnWeight (0.5) making it the richest biome for resources. Mountain has the lowest (0.1) — its power comes from defense and vision, not harvesting. Lava and water have zero spawn weight since neither should contain collectible resources.

Resources

Four resources with different roles: gold is the universal currency, stone is for fortifications, mana powers magic, and food sustains armies. Resource caps prevent hoarding.

  s.resource("gold", {
    color: "#ffd700", rarity: "common",
    maxDeposit: 200, harvestAmount: 15,
  });

  s.resource("stone", {
    color: "#808080", rarity: "uncommon",
    maxDeposit: 150, harvestAmount: 10,
  });

  s.resource("mana", {
    color: "#9c27b0", rarity: "rare",
    maxDeposit: 80, regenRate: 1, harvestAmount: 5,
  });

  s.resource("food", {
    color: "#8bc34a", rarity: "common",
    maxDeposit: 250, regenRate: 3, harvestAmount: 20, decayRate: 1,
  });

  s.resourceCap("gold",  1000);
  s.resourceCap("stone", 500);
  s.resourceCap("mana",  300);
  s.resourceCap("food",  800);
Food decays

Food is the only Castle resource with a decayRate. This forces players to keep building farms and managing supply — you cannot stockpile food indefinitely.

Units

Six unit types cover all roles: frontline melee, ranged damage, magic support, siege, and two neutral threats.

Knight — Frontline Melee

Heavy melee unit with charge ability. 3 experience levels. The backbone of any army.

  s.unit("knight", {
    hp: 100, attack: 18, defense: 14,
    range: 1, speed: 2, visionRange: 3,
    cost: { gold: 50, food: 20 },
    productionTime: 3,
    abilities: ["charge"],
    levels: [
      { level: 2, expRequired: 80,  bonuses: { attack: 4, hp: 20, defense: 2 } },
      { level: 3, expRequired: 200, bonuses: { attack: 8, hp: 40, defense: 5 } },
    ],
    expPerKill: 30,
  });

Archer — Ranged Damage

Fragile ranged unit with volley (hits up to 3 targets in an area). Excellent vision range.

  s.unit("archer", {
    hp: 55, attack: 14, defense: 5,
    range: 3, speed: 2, visionRange: 5,
    cost: { gold: 35, food: 15 },
    productionTime: 2,
    abilities: ["volley"],
    levels: [
      { level: 2, expRequired: 60,  bonuses: { attack: 3, range: 1 } },
      { level: 3, expRequired: 160, bonuses: { attack: 6, visionRange: 1 } },
    ],
    expPerKill: 25,
  });

Mage — Magic Support

Dual-role caster with fireball (AOE fire damage + burn) and healAlly. Requires mana upkeep each turn.

  s.unit("mage", {
    hp: 40, attack: 10, defense: 3,
    range: 3, speed: 1, visionRange: 4,
    cost: { gold: 80, mana: 30 },
    productionTime: 5,
    abilities: ["fireball", "healAlly"],
    upkeep: { mana: 2 },
    levels: [
      { level: 2, expRequired: 100, bonuses: { attack: 5, hp: 10 } },
      { level: 3, expRequired: 250, bonuses: { attack: 10, hp: 20, range: 1 } },
    ],
    expPerKill: 40,
  });

Catapult — Siege Weapon

Slow but devastating. Bombardment deals 60 AOE damage with stun. Essential for breaking walls and castles.

  s.unit("catapult", {
    hp: 70, attack: 25, defense: 4,
    range: 5, speed: 1, visionRange: 2,
    cost: { gold: 100, stone: 60 },
    productionTime: 6,
    abilities: ["bombardment"],
    aoeRadius: 2,
    expPerKill: 50,
  });

Dragon — Neutral Boss

Spawned by the dragon_attack event. Immune to fire, can cross lava. Massive stats and 100 XP on kill.

  s.unit("dragon", {
    hp: 250, attack: 45, defense: 20,
    range: 2, speed: 3, visionRange: 6,
    abilities: ["fireBreath"],
    neutral: true,
    immunities: ["fire"],
    expPerKill: 100,
  });

Skeleton — Neutral Swarm

Weak individually but spawns in groups of 5. Triggered by the undead_uprising event (15 at once).

  s.unit("skeleton", {
    hp: 25, attack: 6, defense: 2,
    range: 1, speed: 2, visionRange: 2,
    neutral: true,
    spawnGroup: 5,
    expPerKill: 10,
  });

Buildings

Eight building types covering production, defense, economy, and support. Most can be upgraded.

Castle — Main Base (3 Levels)

The primary building. Produces knights and archers. Upgrades increase HP, defense, and vision. Also provides resource storage.

  s.building("castle", {
    hp: 500, defense: 20,
    cost: { gold: 0 },
    produces: ["knight", "archer"],
    visionRange: 6, maxLevel: 3,
    levels: [
      { level: 2, upgradeCost: { gold: 200, stone: 100 },
        bonuses: { hp: 150, defense: 5 } },
      { level: 3, upgradeCost: { gold: 400, stone: 200 },
        bonuses: { hp: 300, defense: 10, visionRange: 2 } },
    ],
    storage: { gold: 500, stone: 200, mana: 100, food: 300 },
  });

Barracks — Military Production

Produces knights, archers, and catapults. Has a productionSpeedBonus of 1 — units train one turn faster here.

  s.building("barracks", {
    hp: 200, defense: 10,
    cost: { gold: 80, stone: 40 },
    produces: ["knight", "archer", "catapult"],
    visionRange: 3, maxLevel: 2,
    levels: [
      { level: 2, upgradeCost: { gold: 120, stone: 60 },
        bonuses: { hp: 80 } },
    ],
    productionSpeedBonus: 1,
  });

Tower — Automated Defense

Auto-attacks enemies within range. Excellent vision (8 hexes). Upgradable attack and range.

  s.building("tower", {
    hp: 150, defense: 12,
    cost: { gold: 60, stone: 50 },
    visionRange: 8,
    autoAttack: true, attack: 15, range: 4,
    maxLevel: 3,
    levels: [
      { level: 2, upgradeCost: { gold: 80, stone: 40 },
        bonuses: { attack: 5, range: 1 } },
      { level: 3, upgradeCost: { gold: 150, stone: 80 },
        bonuses: { attack: 10, range: 1, hp: 50 } },
    ],
  });

Tavern — Morale Aura

Emits a morale_boost aura in a 4-hex radius, giving +10 morale to nearby friendly units.

  s.building("tavern", {
    hp: 100, defense: 5,
    cost: { gold: 60, food: 30 },
    visionRange: 3,
    aura: "morale_boost", auraRadius: 4,
  });

Mine, Academy, Wall, Farm

Mine (mountain-only, passive gold). Academy (produces mages, exp aura). Wall (blocks enemies, allies pass). Farm (plains-only, passive food).

  s.building("mine", {
    hp: 120, defense: 8,
    cost: { gold: 40, stone: 20 },
    mustBePlacedOn: "mountain",
    passiveHarvest: { gold: 5 },
  });

  s.building("academy", {
    hp: 180, defense: 8,
    cost: { gold: 120, stone: 60, mana: 20 },
    produces: ["mage"],
    aura: "exp_boost", auraRadius: 5,
  });

  s.building("wall", {
    hp: 250, defense: 25,
    cost: { stone: 30 },
    passable: false, ownTeamPassable: true,
  });

  s.building("farm", {
    hp: 80, defense: 3,
    cost: { gold: 30 },
    mustBePlacedOn: "plains",
    passiveHarvest: { food: 8 },
  });

Events

Seven random events keep the game dynamic. Each has a chance per turn, a minimum step before it can trigger, and a maximum number of occurrences per game.

Event Chance Min Step Max Effect
dragon_attack 2% 20 5 Spawns a neutral dragon on a mountain hex
undead_uprising 0.5% 50 3 Spawns 15 neutral skeletons across the map
gold_rush 3% 10 6 Adds 60 gold to 4 random mountain hexes
plague 1% 30 2 Applies poison to all units on the map
magical_storm 1.5% 25 4 Sets 6 random forest hexes on fire
holy_ground 2% 15 5 Heals and boosts morale on 3 plains hexes
tournament 1% 40 2 Double score multiplier for 20 turns 
  s.event("dragon_attack", {
    chance: 0.02, minStep: 20, maxPerGame: 5,
    message: "A fearsome dragon descends from the mountains!",
    effect: e => {
      const cell = e.randomCell({ biome: "mountain" });
      e.spawnNeutral("dragon", cell);
    },
  });

  s.event("undead_uprising", {
    chance: 0.005, minStep: 50, maxPerGame: 3,
    message: "The dead rise from their graves! Skeletons swarm the land!",
    effect: e => {
      e.spawnNeutralGroup("skeleton", 15);
    },
  });

Map Generation

Biome weights: 40% plains, 25% forest, 20% mountain, 10% water, 5% lava. Forest and mountain form clusters. Two rivers cut across the map to create natural borders.

  s.mapgen(g => {
    g.biome("plains",   0.40);
    g.biome("forest",   0.25);
    g.biome("mountain", 0.20);
    g.biome("water",    0.10);
    g.biome("lava",     0.05);

    g.scatter("gold",  { biome: "mountain", density: 0.08 });
    g.scatter("stone", { biome: "mountain", density: 0.06 });
    g.scatter("mana",  { biome: "forest",   density: 0.03 });
    g.scatter("food",  { biome: "plains",   density: 0.10 });

    g.clusters("forest",   { minSize: 4, maxSize: 10, count: 6 });
    g.clusters("mountain", { minSize: 3, maxSize: 7,  count: 4 });

    g.rivers({ count: 2, minLength: 6, maxLength: 15 });

    g.clearAroundStart(3);
  });
Rivers as natural borders

The two rivers use the water biome (impassable). They force players to find chokepoints and bridges, adding strategic depth to army movement and territory control.

Start Conditions and Victory

Each player begins with 100 gold, 50 stone, 80 food, one knight, a castle at the start position, and a barracks 2 hexes away. Victory requires controlling 70% of the map (domination).

  s.startCondition(p => {
    p.give("gold", 100);
    p.give("stone", 50);
    p.give("food", 80);
    p.spawnUnit("knight", p.startPos);
    p.buildBuilding("castle", p.startPos);
    p.buildBuilding("barracks", p.nearPos(2));
  });

  s.victoryCondition("domination", { percent: 70 });

Warm medieval browns with a gold accent. The MedievalSharp font reinforces the fantasy setting. Soft fog edges create an atmospheric feel.

  s.uiTheme({
    font:           "MedievalSharp",

    secondaryColor: "#4e342e",
    accentColor:    "#ffab00",
    fogColor:       "#1a1a1a",
    fogOpacity:     0.75,
    fogEdge:        "soft",
    minimapBg:      "#2c1810",
  });

Cyberpunk: Scorched Network

A post-apocalyptic sci-fi scenario with drones, combat robots, hackers, and rogue AI. Players scavenge a ruined world for resources, build tech infrastructure, and compete for points. First to 1000 score wins.

Meta and Rules Setup

The Cyberpunk scenario uses a different rule set. Logistics mode is on — units must physically carry resources back to base. Morale is off because robots do not have feelings. No day/night cycle or weather in a scorched wasteland.

scenario("cyberpunk", s => {

  s.name("Cyberpunk: Scorched Network");
  s.description("Post-apocalyptic warfare with robots, hackers, and rogue AI.");
  s.version("1.0.0");
  s.author("CodeHex Team");

  s.rules({
    friendlyFire:       false,
    logisticsMode:      true,
    moraleSystem:       false,
    experienceSystem:   true,
    resourceCaps:       true,
    globalResourcePool: false,
    nightCycle:         false,
    weather:            false,
    siegeMode:          true,
  });
Logistics mode changes everything

With logisticsMode: true, players need drones or other units with carryCap to physically transport resources from deposits back to buildings with storage. This creates supply-line gameplay that the Castle scenario does not have.

Biomes

Five biomes represent a destroyed urban landscape. Wasteland is open ground, ruins provide cover, toxic zones punish careless movement, data streams boost tech units, and bunkers are fortified positions.

  s.biome("wasteland", {
    color: "#a0826d", moveCost: 1,
    defenseBonus: 0, spawnWeight: 0.2,
  });

  s.biome("ruins", {
    color: "#6d6d6d", moveCost: 2,
    defenseBonus: 15, attackBonus: 5,
    visionPenalty: 1, spawnWeight: 0.4,
  });

  s.biome("toxic_zone", {
    color: "#7cb342", moveCost: 2,
    damagePerTurn: 5, spawnWeight: 0.6,
  });

  s.biome("data_stream", {
    color: "#00bcd4", moveCost: 1,
    harvestBonus: 25, visionBonus: 1, spawnWeight: 0.0,
  });

  s.biome("bunker", {
    color: "#455a64", moveCost: 1,
    defenseBonus: 40, visionPenalty: 2, spawnWeight: 0.0,
  });
Design rationale

Toxic zones have the highest spawnWeight (0.6) — rare resources like rare_earth appear there, forcing players to risk radiation damage for valuable materials. Bunkers offer extreme defense (+40) but reduce vision by 2, creating a risk/reward trade-off for turtling strategies. Data streams are clean pathways with harvest bonuses, making them valuable territory to control.

Resources

Four resources: metal is the base currency, energy powers everything (but decays), rare earth is needed for advanced units, and data unlocks tech buildings.

  s.resource("metal", {
    color: "#b0bec5", rarity: "common",
    maxDeposit: 200, harvestAmount: 12,
  });

  s.resource("energy", {
    color: "#ffeb3b", rarity: "common",
    maxDeposit: 150, regenRate: 2,
    harvestAmount: 10, decayRate: 1,
  });

  s.resource("rare_earth", {
    color: "#e040fb", rarity: "rare",
    maxDeposit: 60, harvestAmount: 3,
  });

  s.resource("data", {
    color: "#00e5ff", rarity: "uncommon",
    maxDeposit: 100, regenRate: 1, harvestAmount: 8,
  });

  s.resourceCap("metal",      800);
  s.resourceCap("energy",     500);
  s.resourceCap("rare_earth", 200);
  s.resourceCap("data",       400);
Energy has decay

Energy is the only Cyberpunk resource with a decayRate of 1. Combined with logistics mode, this means energy must be harvested, transported, and spent quickly. Generators help offset the decay with passive income.

Units

Six unit types: a fast scout, frontline combatant, tech disruptor, heavy assault, and two neutral threats.

Drone — Fast Scout

Speed 4, vision 7, carryCap: 20. The primary logistics unit — fast enough to ferry resources between deposits and base.

  s.unit("drone", {
    hp: 30, attack: 6, defense: 2,
    range: 1, speed: 4, visionRange: 7,
    cost: { metal: 20, energy: 10 },
    productionTime: 1,
    carryCap: 20,
    levels: [
      { level: 2, expRequired: 40,  bonuses: { speed: 1, visionRange: 1 } },
      { level: 3, expRequired: 100, bonuses: { speed: 1, visionRange: 2, hp: 10 } },
    ],
    expPerKill: 10,
  });

Combat Robot — Frontline (3 Levels)

Balanced stats with overclock ability (2.5x damage burst). Levels into a formidable fighter at level 3 with +1 range.

  s.unit("combat_robot", {
    hp: 90, attack: 16, defense: 12,
    range: 2, speed: 2, visionRange: 4,
    cost: { metal: 60, energy: 30 },
    productionTime: 3,
    abilities: ["overclock"],
    carryCap: 10,
    levels: [
      { level: 2, expRequired: 80,  bonuses: { attack: 4, defense: 3, hp: 15 } },
      { level: 3, expRequired: 200, bonuses: { attack: 8, defense: 6, hp: 30, range: 1 } },
    ],
    expPerKill: 35,
  });

Hacker — Tech Disruptor

Two abilities: emp (AOE disable) and dataSteal (shuts down enemy buildings). Requires energy upkeep each turn.

  s.unit("hacker", {
    hp: 35, attack: 5, defense: 3,
    range: 4, speed: 2, visionRange: 5,
    cost: { metal: 40, data: 25 },
    productionTime: 4,
    abilities: ["emp", "dataSteal"],
    upkeep: { energy: 1 },
    levels: [
      { level: 2, expRequired: 70,  bonuses: { range: 1, hp: 10 } },
      { level: 3, expRequired: 180, bonuses: { range: 1, visionRange: 2 } },
    ],
    expPerKill: 30,
  });

Colossus — Heavy AOE

Massive unit with plasmaBeam (45 energy damage). Costs rare earth to build. Energy upkeep of 3 per turn.

  s.unit("colossus", {
    hp: 200, attack: 40, defense: 18,
    range: 2, speed: 1, visionRange: 3,
    cost: { metal: 150, energy: 80, rare_earth: 20 },
    productionTime: 8,
    abilities: ["plasmaBeam"],
    aoeRadius: 1,
    upkeep: { energy: 3 },
    levels: [
      { level: 2, expRequired: 150, bonuses: { attack: 10, hp: 40 } },
      { level: 3, expRequired: 350, bonuses: { attack: 20, hp: 80, defense: 5 } },
    ],
    expPerKill: 80,
  });

Rogue AI — Neutral Threat

Spawned by the rogue_uprising event. Has the emp ability and is immune to energy damage.

  s.unit("rogue_ai", {
    hp: 120, attack: 22, defense: 10,
    range: 3, speed: 3, visionRange: 6,
    abilities: ["emp"],
    neutral: true,
    immunities: ["energy"],
    expPerKill: 60,
  });

Mutant — Neutral Toxic Swarm

Spawns in groups of 4. Immune to radiation, making them a persistent threat in toxic zones.

  s.unit("mutant", {
    hp: 45, attack: 10, defense: 4,
    range: 1, speed: 2, visionRange: 3,
    neutral: true,
    spawnGroup: 4,
    immunities: ["radiation"],
    expPerKill: 15,
  });

Buildings

Seven building types with a focus on tech infrastructure and passive resource generation.

Command Center — Universal Production

Unlike the Castle scenario's split production, the command center produces all four unit types. 3 upgrade levels.

  s.building("command_center", {
    hp: 400, defense: 18,
    cost: { metal: 0 },
    produces: ["drone", "combat_robot", "hacker", "colossus"],
    visionRange: 6, maxLevel: 3,
    levels: [
      { level: 2, upgradeCost: { metal: 200, energy: 100 },
        bonuses: { hp: 100, defense: 5 } },
      { level: 3, upgradeCost: { metal: 400, energy: 200, rare_earth: 30 },
        bonuses: { hp: 200, defense: 10, visionRange: 2 } },
    ],
    storage: { metal: 400, energy: 200, rare_earth: 50, data: 200 },
  });

Factory — Robot Production

Specialized for combat robots and colossi. Production speed bonus reduces training time.

  s.building("factory", {
    hp: 250, defense: 12,
    cost: { metal: 100, energy: 50 },
    produces: ["combat_robot", "colossus"],
    visionRange: 3, maxLevel: 3,
    productionSpeedBonus: 1,
  });

Generator — Passive Energy

Produces 6 energy per turn passively, countering the energy decay mechanic. Upgradeable for +4 more.

  s.building("generator", {
    hp: 150, defense: 8,
    cost: { metal: 60 },
    passiveHarvest: { energy: 6 },
    maxLevel: 2,
    levels: [
      { level: 2, upgradeCost: { metal: 80, rare_earth: 10 },
        bonuses: { passiveHarvest: { energy: 4 } } },
    ],
  });

Server Tower, Shield Dome, Radar, Recycler

Server tower (auto-attack + network aura). Shield dome (defense aura). Radar (vision 12). Recycler (ruins-only, passive metal).

  s.building("server_tower", {
    hp: 200, defense: 10,
    cost: { metal: 80, data: 30 },
    visionRange: 5,
    autoAttack: true, attack: 12, range: 4,
    aura: "network_boost", auraRadius: 4,
  });

  s.building("shield_dome", {
    hp: 180, defense: 20,
    cost: { metal: 100, energy: 60 },
    aura: "shield_aura", auraRadius: 3,
  });

  s.building("radar", {
    hp: 80, defense: 4,
    cost: { metal: 50, energy: 30 },
    visionRange: 12,
  });

  s.building("recycler", {
    hp: 120, defense: 6,
    cost: { metal: 40, energy: 20 },
    mustBePlacedOn: "ruins",
    passiveHarvest: { metal: 4 },
  });

Events

Seven events themed around radiation, technology, and scarcity.

Event Chance Min Step Max Effect
radioactive_fallout 2% 10 6 Places radiation overlays on 5 random hexes
network_virus 1.5% 20 4 Disables ~30% of all units randomly
emp_pulse 1% 30 3 Disables all units on the map
data_cache 3% 8 8 Adds 40 data to 3 data stream hexes
rogue_uprising 0.8% 40 3 Spawns 3 neutral rogue AI units
power_surge 2% 15 4 Generators produce double energy for 10 turns
meteor_strike 1% 25 5 Creates a long-lasting radiation zone at impact 
  s.event("radioactive_fallout", {
    chance: 0.02, minStep: 10, maxPerGame: 6,
    message: "Radioactive fallout detected! Radiation zones are expanding!",
    effect: e => {
      const cells = e.randomCells(5);
      cells.forEach(c => e.addOverlay(c, "radiation", { duration: 12 }));
    },
  });

  s.event("rogue_uprising", {
    chance: 0.008, minStep: 40, maxPerGame: 3,
    message: "Rogue AI units are emerging from the network!",
    effect: e => {
      e.spawnNeutralGroup("rogue_ai", 3);
    },
  });

Map Generation

Biome weights: 35% wasteland, 30% ruins, 15% toxic zone, 10% data stream, 10% bunker. Lots of ruins clusters, no rivers. Three neutral recyclers are pre-placed in ruins.

  s.mapgen(g => {
    g.biome("wasteland",   0.35);
    g.biome("ruins",       0.30);
    g.biome("toxic_zone",  0.15);
    g.biome("data_stream", 0.10);
    g.biome("bunker",      0.10);

    g.scatter("metal",      { biome: "ruins",       density: 0.08 });
    g.scatter("energy",     { biome: "wasteland",   density: 0.06 });
    g.scatter("rare_earth", { biome: "toxic_zone",  density: 0.04 });
    g.scatter("data",       { biome: "data_stream", density: 0.10 });

    g.clusters("ruins",      { minSize: 5, maxSize: 12, count: 7 });
    g.clusters("toxic_zone", { minSize: 3, maxSize: 6,  count: 4 });
    g.clusters("bunker",     { minSize: 1, maxSize: 3,  count: 5 });

    g.clearAroundStart(3);

    g.placeNeutralBuilding("recycler", { count: 3, biome: "ruins" });
  });
No rivers, lots of ruins

Unlike Castle, Cyberpunk has no water biome and no rivers. Movement is never fully blocked by terrain. Instead, toxic zones and radiation overlays create soft barriers that players can push through at a cost. The 7 ruins clusters (up to 12 hexes each) create dense urban areas rich with metal deposits.

Start Conditions and Victory

Each player starts with 200 metal, 100 energy, a drone, a combat robot, and a command center. Victory is by score — first to 1000 points.

  s.startCondition(p => {
    p.give("metal", 200);
    p.give("energy", 100);
    p.spawnUnit("drone", p.nearPos(1));
    p.spawnUnit("combat_robot", p.nearPos(1));
    p.buildBuilding("command_center", p.startPos);
  });

  s.victoryCondition("score", { target: 1000 });

UI Theme

Dark background with a neon green/cyan accent. Share Tech Mono gives a terminal-like feel. Hard fog edges reinforce the digital aesthetic.

  s.uiTheme({
    font:           "Share Tech Mono",
    primaryColor:   "#0d1117",
    secondaryColor: "#161b22",

Key Mechanic Differences

Morale vs. No Morale

Castle uses moraleSystem: true with taverns providing morale_boost auras. This adds a soft support layer — armies near taverns fight harder. Cyberpunk disables morale entirely since robots do not have feelings, shifting the focus to raw stats and abilities.

Auto-Collect vs. Logistics

Castle uses logisticsMode: false — resources go straight to the player pool. Cyberpunk enables it, requiring drones with carryCap: 20 to ferry resources. This alone creates a completely different early-game loop.

Impassable Terrain vs. Soft Barriers

Castle has water (fully impassable) and rivers that hard-block movement. Cyberpunk has no impassable biomes — toxic zones deal damage but can always be crossed, creating a more fluid map.

Territory vs. Points

Castle's domination victory encourages expanding and holding land. Cyberpunk's score victory encourages aggressive play, raiding, and capturing data caches — you do not need to hold territory, just score faster than your opponent.

Same Functions, Different Worlds

The power of the DSL is that the same function calls produce radically different experiences based on the values you pass. Here is a direct comparison:

Rules: same function, different flags

// Castle — atmospheric simulation
s.rules({
  moraleSystem: true, nightCycle: true,
  weather: true, logisticsMode: false,
});

// Cyberpunk — mechanical efficiency
s.rules({
  moraleSystem: false, nightCycle: false,
  weather: false, logisticsMode: true,
});

Hazard biomes: fire immunity vs. radiation immunity

// Castle — lava with dragon exception
s.biome("lava", {
  damagePerTurn: 10,
  allowedUnits: ["dragon"],
});

// Cyberpunk — toxic zone (no exceptions, mutants immune via unit def)
s.biome("toxic_zone", {
  damagePerTurn: 5,
  spawnWeight: 0.6,  // richest biome for resources
});

Support units: healer vs. hacker

// Castle — mage heals allies and burns enemies
s.unit("mage", {
  abilities: ["fireball", "healAlly"],
  upkeep: { mana: 2 },
});

// Cyberpunk — hacker disables enemies and steals data
s.unit("hacker", {
  abilities: ["emp", "dataSteal"],
  upkeep: { energy: 1 },
});

Victory: domination vs. score

// Castle — hold 70% of the map
s.victoryCondition("domination", { percent: 70 });

// Cyberpunk — reach 1000 points
s.victoryCondition("score", { target: 1000 });
