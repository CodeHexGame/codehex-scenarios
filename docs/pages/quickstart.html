Quick Start

Get your first scenario running in 5 minutes.

Prerequisites

All you need is the CodeHex server installed and ready to run. The server includes the template scenario and all tooling required to load custom content.

No build tools, compilers, or additional dependencies are required. Scenarios are plain JavaScript files loaded directly by the server at startup.

Steps

Copy the template

The repository ships with a fully commented template/ directory that works out of the box. Start by copying it:

cp -r template/ my_scenario/

This gives you a complete scenario.js with every DSL function stubbed out and documented.

Edit the scenario file

Open my_scenario/scenario.js in any editor and start customizing. Here is a minimal working scenario with all the essentials:

scenario("my_scenario", s => {

  // Give your scenario a display name
  s.name("My First Scenario");

  // Define at least one biome
  s.biome("grassland", {
    color: "#7ec850",
    moveCost: 1
  });

  // Add a resource players can collect
  s.resource("gold", {
    color: "#ffd700",
    rarity: "common"
  });

  // Create a basic unit
  s.unit("soldier", {
    hp: 50,
    attack: 10,
    defense: 5,
    cost: { gold: 20 },
    color: "#e53935"
  });

  // Place a production building
  s.building("barracks", {
    hp: 200,
    cost: { gold: 60 },
    produces: ["soldier"],
    color: "#8d6e63"
  });

  // Configure map generation
  s.mapgen({
    size: 48,
    biomes: {
      grassland: { weight: 1.0 }
    }
  });

  // Starting conditions for each player
  s.startCondition({
    resources: { gold: 100 },
    units: ["soldier"],
    buildings: ["barracks"]
  });

  // How a player wins the game
  s.victoryCondition({
    type: "domination",
    threshold: 0.7
  });

});

You can add more biomes, units, buildings, abilities, and events at any time. See the DSL Reference for every available function and parameter.

Copy to the server

Move your finished scenario folder into the server's scenarios/ directory:

cp -r my_scenario/ codehex-server/scenarios/my_scenario/

Set the active scenario in config.json

Open the server's config.json and point it at your new scenario:

{ "scenario": "my_scenario" }

Start the server and verify

Launch the CodeHex server:

npm start

Watch the console output. You should see a log line confirming your scenario was loaded:

[CodeHex] Scenario "my_scenario" loaded successfully.

If you see an error instead, double-check that your scenario.js has no syntax errors and that the folder name in config.json matches the directory you copied.
