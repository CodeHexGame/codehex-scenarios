Buildings

Define structures that produce units, harvest resources, defend territory, and buff nearby allies. Buildings are created with s.building(id, options).

All Parameters

Every property you can pass in the options object when calling s.building().

Parameter Type Default Description
hp number 100 Hit points. When reduced to zero the building is destroyed.
defense number 0 Flat damage reduction applied to every incoming attack.
size number 1 Number of hex tiles the building occupies (1 = single tile).
cost object {} Resource cost to construct. Keys are resource IDs, values are amounts (e.g. { gold: 80, wood: 40 }).
produces string[] [] Array of unit IDs this building can produce.
visionRange number 2 Fog-of-war sight radius in hex tiles.
aura object null An aura effect object created with s.aura(). Applied to friendly units within auraRadius.
auraRadius number 0 Radius in tiles for the aura. Only meaningful when aura is set.
auraEffect string null Optional visual identifier for the aura overlay shown on the map.
storage object null Resource storage caps added when this building exists. Keys are resource IDs, values are capacity amounts (e.g. { gold: 500 }).
maxLevel number 1 Maximum upgrade level. Set to 1 for buildings that cannot be upgraded.
levels array [] Array of upgrade definitions. Each entry is an object with upgradeCost and stat bonuses (see Upgrade System).
passiveHarvest object null Resources generated automatically each turn. Keys are resource IDs, values are per-turn amounts.
mustBePlacedOn string null Biome ID that this building must be placed on. Construction is blocked on any other terrain.
passable boolean true Whether any unit can move through the building tile.
ownTeamPassable boolean true Whether the owning team's units can move through the tile, even when passable is false.
autoAttack boolean false When true, the building automatically attacks the nearest enemy in range each turn.
attack number 0 Damage dealt per auto-attack. Only used when autoAttack is true.
range number 0 Attack range in hex tiles. Only used when autoAttack is true.
productionSpeedBonus number 0 Percentage bonus to unit production speed (e.g. 0.25 = 25% faster). Stacks additively with other bonuses.

Building Types

Buildings fall into four broad categories depending on which parameters you set. A single building can combine multiple roles.

Production

Uses produces to train units. Optionally boosted by productionSpeedBonus.

Defensive

Sets autoAttack: true with attack and range to fire on enemies automatically.

Economic

Uses passiveHarvest to generate resources every turn without player input.

Support

Provides an aura that buffs friendly units within auraRadius.

Upgrade System

Set maxLevel higher than 1, then provide a levels array. Each entry after the base level defines the cost and stat bonuses for that upgrade tier. The index in the array corresponds to the level (index 0 = level 2 upgrade, index 1 = level 3 upgrade, etc.).

s.building("castle", {
  hp: 500,
  defense: 10,
  maxLevel: 3,
  levels: [
    { upgradeCost: { gold: 200 }, hp: 200, defense: 5 },
    { upgradeCost: { gold: 500 }, hp: 400, defense: 10, visionRange: 1 }
  ]
});

In this example the castle starts at level 1 with 500 HP and 10 defense. Upgrading to level 2 costs 200 gold and adds 200 HP and 5 defense. Level 3 costs 500 gold and adds another 400 HP, 10 defense, and 1 extra vision range.

Every numeric building stat can appear as a bonus in a levels entry. The bonus value is added to the current stat, not replaced.

Passive Harvest

Buildings with passiveHarvest automatically deposit resources into the owning player's stockpile at the start of each turn. No unit interaction is required.

s.building("mine", {
  hp: 150,
  cost: { gold: 50, wood: 30 },
  passiveHarvest: { gold: 10 },
  mustBePlacedOn: "mountain"
});

The mine generates 10 gold per turn but can only be built on mountain tiles. If the player's storage for gold is full, excess resources are lost.

Combine passiveHarvest with storage on the same building to increase the player's resource cap alongside the income stream.

Auras

A building aura applies a persistent effect to all friendly units within range. Create the aura effect with s.aura() and assign it to the building's aura property.

s.building("tavern", {
  hp: 120,
  cost: { gold: 80 },
  aura: s.aura({
    effect: { morale: 5 }
  }),
  auraRadius: 3,
  auraEffect: "golden_glow"
});

Every friendly unit within 3 tiles of the tavern receives +5 morale. The auraEffect string references a visual overlay so players can see the radius on the map.

Aura bonuses are recomputed each turn. A unit that moves out of range immediately loses the bonus. See Abilities — Auras for the full s.aura() API.

AutoAttack Towers

Defensive buildings fire automatically at the nearest enemy within range at the start of each combat phase. Set autoAttack: true and provide attack and range.

s.building("tower", {
  hp: 200,
  defense: 5,
  cost: { gold: 100, stone: 60 },
  autoAttack: true,
  attack: 25,
  range: 3,
  maxLevel: 3,
  levels: [
    { upgradeCost: { gold: 80 },  attack: 10, range: 1 },
    { upgradeCost: { gold: 160 }, attack: 15, hp: 100 }
  ]
});

This tower starts with 25 attack and 3 range. At level 2 it gains +10 attack and +1 range. At level 3 it gains +15 attack and +100 HP.

AutoAttack buildings always target the nearest enemy. There is no way to manually choose a target. If you need player-controlled targeting, use a unit with an ability instead.

Placement Restrictions

Use mustBePlacedOn to restrict a building to a specific biome. The value is a biome ID defined with s.biome(). If the player tries to place the building on any other terrain, construction is blocked.

s.building("mine", {
  hp: 150,
  cost: { gold: 50 },
  passiveHarvest: { gold: 10 },
  mustBePlacedOn: "mountain"
});

s.building("farm", {
  hp: 80,
  cost: { gold: 30 },
  passiveHarvest: { food: 5 },
  mustBePlacedOn: "plains"
});

Placement restrictions add strategic depth by forcing players to compete over specific terrain types. See Biomes for how to configure terrain distribution.

Passability

By default, all buildings are passable — units can walk through their tile. Two flags control movement:

  • passable: false — No unit from any team can move through this tile.
  • ownTeamPassable: true — Even when passable is false, the owning team's units can still pass through.
s.building("wall", {
  hp: 400,
  defense: 20,
  cost: { stone: 40 },
  passable: false,
  ownTeamPassable: false
});

Walls block all movement including the owner's units. This is useful for creating defensive chokepoints.

Storage

The storage property increases the player's maximum resource capacity while the building stands. If the building is destroyed, the cap decreases and any excess resources are lost.

s.building("warehouse", {
  hp: 120,
  cost: { gold: 60, wood: 40 },
  storage: { gold: 500, wood: 300, food: 200 }
});

Castle Scenario Examples

The official Castle scenario demonstrates all major building archetypes.

Castle (Main Building)

The player's headquarters. Three upgrade levels that increase HP, defense, and vision range.

s.building("castle", {
  hp: 500,
  defense: 10,
  size: 2,
  visionRange: 4,
  produces: ["peasant"],
  storage: { gold: 1000, wood: 500, stone: 500, food: 300 },
  maxLevel: 3,
  levels: [
    { upgradeCost: { gold: 300, stone: 200 }, hp: 200, defense: 5, visionRange: 1 },
    { upgradeCost: { gold: 600, stone: 400 }, hp: 300, defense: 10, visionRange: 1 }
  ]
});

Barracks (Production Speed)

Trains military units. The productionSpeedBonus makes units ready faster.

s.building("barracks", {
  hp: 250,
  defense: 5,
  cost: { gold: 80, wood: 60 },
  produces: ["soldier", "archer", "knight"],
  productionSpeedBonus: 0.25
});

Tower (AutoAttack, 3 Levels)

s.building("tower", {
  hp: 200,
  defense: 5,
  cost: { gold: 100, stone: 60 },
  autoAttack: true,
  attack: 25,
  range: 3,
  maxLevel: 3,
  levels: [
    { upgradeCost: { gold: 80, stone: 40 },  attack: 10, range: 1 },
    { upgradeCost: { gold: 160, stone: 80 }, attack: 15, hp: 100 }
  ]
});

Tavern (Morale Aura)

s.building("tavern", {
  hp: 120,
  cost: { gold: 80, wood: 40 },
  aura: s.aura({
    effect: { morale: 5, attack: 2 }
  }),
  auraRadius: 3,
  auraEffect: "golden_glow"
});

Mine (Mountain Only, Gold)

s.building("mine", {
  hp: 150,
  cost: { gold: 50, wood: 30 },
  passiveHarvest: { gold: 10 },
  mustBePlacedOn: "mountain",
  storage: { gold: 200 }
});

Academy (Mage Production, EXP Aura)

s.building("academy", {
  hp: 180,
  defense: 3,
  cost: { gold: 150, stone: 80 },
  produces: ["mage"],
  aura: s.aura({
    effect: { expGain: 0.5 }
  }),
  auraRadius: 2
});

Wall (Impassable Defense)

s.building("wall", {
  hp: 400,
  defense: 20,
  cost: { stone: 40 },
  passable: false,
  ownTeamPassable: false
});

Farm (Plains, Food)

s.building("farm", {
  hp: 80,
  cost: { gold: 30, wood: 20 },
  passiveHarvest: { food: 5 },
  mustBePlacedOn: "plains",
  storage: { food: 100 }
});

Cyberpunk Scenario Examples

The Cyberpunk scenario uses buildings with futuristic themes and different resource types.

Command Center

The faction HQ. Produces basic drones and stores all three resource types.

s.building("command_center", {
  hp: 600,
  defense: 15,
  size: 2,
  visionRange: 5,
  produces: ["drone"],
  storage: { credits: 2000, energy: 500, data: 500 },
  maxLevel: 3,
  levels: [
    { upgradeCost: { credits: 500, data: 200 }, hp: 200, defense: 5, visionRange: 1 },
    { upgradeCost: { credits: 1000, data: 500 }, hp: 300, defense: 10, visionRange: 2 }
  ]
});

Factory

Produces combat units with a production speed bonus.

s.building("factory", {
  hp: 300,
  defense: 8,
  cost: { credits: 200, energy: 100 },
  produces: ["mech", "cyborg", "hacker"],
  productionSpeedBonus: 0.3
});

Generator

Passive energy production each turn.

s.building("generator", {
  hp: 150,
  cost: { credits: 120 },
  passiveHarvest: { energy: 15 },
  storage: { energy: 300 }
});

Server Tower

Passive data generation with an EXP aura for nearby units.

s.building("server_tower", {
  hp: 120,
  defense: 3,
  cost: { credits: 180, energy: 80 },
  passiveHarvest: { data: 8 },
  aura: s.aura({
    effect: { expGain: 0.3 }
  }),
  auraRadius: 3
});

Shield Dome

Defensive aura that increases nearby units' defense.

s.building("shield_dome", {
  hp: 200,
  defense: 10,
  cost: { credits: 250, energy: 150 },
  aura: s.aura({
    effect: { defense: 8 }
  }),
  auraRadius: 4,
  auraEffect: "blue_shield"
});

Radar

Long-range vision building with an aura that increases unit sight.

s.building("radar", {
  hp: 100,
  cost: { credits: 140, data: 60 },
  visionRange: 8,
  aura: s.aura({
    effect: { visionRange: 2 }
  }),
  auraRadius: 3
});

Recycler

Converts wreckage into credits. Harvests passively and adds storage.

s.building("recycler", {
  hp: 130,
  cost: { credits: 100, energy: 50 },
  passiveHarvest: { credits: 12 },
  storage: { credits: 400 }
});
