Abilities

Give units active skills that deal damage, heal allies, apply status effects, and more. Abilities are defined with s.ability(id, options) and assigned to units via their abilities array.

All Parameters

Every property you can pass in the options object when calling s.ability().

Parameter Type Default Description
cooldown number 1 Turns the unit must wait between uses of this ability.
range number 1 Maximum distance in hex tiles to the target.
damage number 0 Base damage dealt to the target.
damageMulti number 1.0 Multiplier applied to the base damage (e.g. 1.5 = 150% damage).
aoeRadius number 0 Area-of-effect radius. When greater than 0, the ability hits all valid targets within this radius of the target tile.
cost object {} Resource cost to use the ability. Keys are resource IDs, values are amounts.
damageType string "physical" The type of damage dealt. Interacts with unit resistances. See Damage Types.
effect object null A status effect to apply on hit. Created with s.statusEffect(). See Status Effects.
effectDuration number 0 Number of turns the status effect lasts. Only meaningful when effect is set.
targetsCount number 1 Number of targets the ability hits. Targets are chosen by proximity to the aim point.
targetType string "enemy" Who can be targeted: "enemy", "ally", "building", or "any".
healAmount number 0 HP restored to the target. Used for healing abilities (typically with targetType: "ally").

Damage Types

Every damaging ability has a damageType that determines how it interacts with unit resistances. The engine ships with five built-in types, but you can define any custom string.

Type Description
physicalStandard weapon damage. Reduced by the target's defense stat.
fireElemental fire. Often paired with a burn status effect.
poisonToxic damage. Commonly applies damage-over-time effects.
energySci-fi energy weapons. Ignores a portion of physical defense.
radiationArea denial damage. Often used with overlays.

Custom damage types are allowed. Just use any string — for example damageType: "arcane". Pair it with a matching resistance key on your units to create unique damage interactions.

AOE Attacks

When aoeRadius is greater than 0, the ability targets a hex tile rather than a single unit. All valid targets (based on targetType) within the radius take the full damage.

s.ability("fireball", {
  damage: 30,
  damageType: "fire",
  range: 4,
  aoeRadius: 2,
  cooldown: 3,
  effect: s.statusEffect({
    damagePerTurn: 5,
    damageType: "fire",
    duration: 2,
    color: "#ff4500"
  }),
  effectDuration: 2
});

The fireball hits every enemy within 2 tiles of the target for 30 fire damage and applies a burn that deals 5 fire damage per turn for 2 turns.

AOE abilities cannot distinguish between targets within the radius. If targetType is "any", friendly units will also take damage. Plan AOE ranges carefully.

Cooldown System

Cooldowns are tracked per unit instance, not globally. When a unit uses an ability, that ability becomes unavailable for cooldown turns. A cooldown of 1 means the ability can be used every turn. A cooldown of 3 means two turns must pass after use before the ability is available again.

// Available every turn
s.ability("slash", { damage: 15, cooldown: 1 });

// Available every 3rd turn
s.ability("power_strike", { damage: 45, cooldown: 3 });

Cooldowns start counting after the turn the ability is used. A cooldown of 2 means: use on turn 1, skip turn 2, available again on turn 3.

Status Effects

Create status effects with s.statusEffect() and attach them to abilities via the effect parameter. When the ability hits, the effect is applied to the target for effectDuration turns.

s.statusEffect() Parameters

Parameter Type Default Description
duration number 1 How many turns the effect persists.
blocksMove boolean false Prevents the affected unit from moving.
blocksAttack boolean false Prevents the affected unit from attacking or using abilities.
damagePerTurn number 0 Damage dealt to the unit at the start of each turn.
damageType string "physical" Damage type for damagePerTurn. Interacts with resistances.
color string null Hex color for the visual indicator shown on the affected unit. 
// Stun: blocks both movement and attacks for 1 turn
const stun = s.statusEffect({
  duration: 1,
  blocksMove: true,
  blocksAttack: true,
  color: "#ffff00"
});

// Poison: damage over time for 3 turns
const poison = s.statusEffect({
  duration: 3,
  damagePerTurn: 8,
  damageType: "poison",
  color: "#00ff66"
});

Target Types

The targetType parameter controls what the ability can be aimed at.

Value Valid Targets
"enemy"Opposing team's units only.
"ally"Own team's units (including self).
"building"Any building (friend or foe).
"any"Any unit or building regardless of team.

Multi-Target Abilities

Set targetsCount to hit more than one target per use. The engine selects the closest valid targets to the aim point. This is different from AOE — multi-target hits specific units, while AOE hits an area.

s.ability("volley", {
  damage: 15,
  range: 5,
  targetsCount: 3,
  aoeRadius: 1,
  cooldown: 2,
  targetType: "enemy"
});

Volley selects up to 3 enemies within range and hits each one plus all enemies within 1 tile of each target.

Healing Abilities

Use healAmount with targetType: "ally" to create healing abilities. Healing abilities can also have AOE and multi-target.

s.ability("healAlly", {
  healAmount: 30,
  range: 3,
  cooldown: 2,
  targetType: "ally"
});

Healing cannot exceed a unit's maximum HP. Over-healing is wasted.

Resource Cost

Powerful abilities can require resources to activate. The player must have sufficient resources or the ability is grayed out.

s.ability("bombardment", {
  damage: 80,
  damageType: "physical",
  range: 6,
  aoeRadius: 2,
  cooldown: 4,
  cost: { gold: 30 },
  targetType: "any"
});

Bombardment costs 30 gold per use, making it a powerful but expensive siege tool.

Auras — s.aura()

Auras are passive effects that radiate from buildings (or units with aura properties). Unlike abilities, auras are always active and do not require activation. Create them with s.aura().

s.aura() Parameters

Parameter Type Default Description
effect object required An object of stat bonuses applied to units in range. Keys are any stat name, values are the bonus amount (e.g. { attack: 5, defense: 3 }).
stackable boolean false When true, multiple instances of the same aura stack. When false, only the strongest instance applies. 
// Non-stacking morale aura
s.aura({
  effect: { morale: 5, attack: 2 },
  stackable: false
});

// Stacking experience aura (multiple academies add up)
s.aura({
  effect: { expGain: 0.5 },
  stackable: true
});

Auras are assigned to buildings via the aura property and take effect within auraRadius tiles. See Buildings — Auras for usage examples.

Overlays — s.overlay()

Overlays are persistent terrain effects placed on hex tiles. They can deal damage, apply effects, and spread to adjacent tiles. Create them with s.overlay().

s.overlay() Parameters

Parameter Type Default Description
color string required Hex color of the overlay rendered on the tile.
opacity number 0.5 Transparency of the overlay (0.0 to 1.0).
damagePerTurn number 0 Damage dealt to any unit standing on the tile each turn.
spreadChance number 0 Probability (0.0 to 1.0) that the overlay spreads to an adjacent tile each turn.
duration number -1 Turns until the overlay disappears. -1 means permanent.
effect object null A status effect (from s.statusEffect()) applied to units that enter the tile. 
// Spreading fire overlay
s.overlay("fire_zone", {
  color: "#ff4500",
  opacity: 0.4,
  damagePerTurn: 10,
  damageType: "fire",
  spreadChance: 0.3,
  duration: 5
});

// Radiation zone with movement slow
s.overlay("radiation", {
  color: "#00ff00",
  opacity: 0.3,
  damagePerTurn: 5,
  damageType: "radiation",
  spreadChance: 0.1,
  duration: -1,
  effect: s.statusEffect({
    blocksMove: false,
    damagePerTurn: 3,
    damageType: "radiation",
    duration: 2,
    color: "#00ff00"
  })
});

Overlays with spreadChance greater than 0 can grow out of control. Always set a finite duration for spreading overlays, or they may eventually cover the entire map.

Castle Scenario Examples

The official Castle scenario showcases a range of medieval-themed abilities.

Charge (Stun)

A knight ability that deals damage and stuns the target for 1 turn.

s.ability("charge", {
  damage: 20,
  damageType: "physical",
  range: 2,
  cooldown: 3,
  effect: s.statusEffect({
    duration: 1,
    blocksMove: true,
    blocksAttack: true,
    color: "#ffff00"
  }),
  effectDuration: 1,
  targetType: "enemy"
});

Volley (Multi-Target AOE)

Archers fire at multiple enemies simultaneously, with a small splash on each target.

s.ability("volley", {
  damage: 15,
  damageType: "physical",
  range: 5,
  cooldown: 2,
  targetsCount: 3,
  aoeRadius: 1,
  targetType: "enemy"
});

Fireball (Fire + Burn)

A mage spell that deals fire damage in an area and leaves a burn on hit targets.

s.ability("fireball", {
  damage: 30,
  damageType: "fire",
  range: 4,
  aoeRadius: 2,
  cooldown: 3,
  effect: s.statusEffect({
    damagePerTurn: 5,
    damageType: "fire",
    duration: 2,
    color: "#ff4500"
  }),
  effectDuration: 2,
  targetType: "enemy"
});

Heal Ally

A cleric ability that restores HP to a friendly unit.

s.ability("healAlly", {
  healAmount: 30,
  range: 3,
  cooldown: 2,
  targetType: "ally"
});

Fire Breath (Dragon AOE)

A dragon's breath weapon. High damage in a cone-like area with a large radius.

s.ability("fireBreath", {
  damage: 40,
  damageType: "fire",
  range: 2,
  aoeRadius: 3,
  cooldown: 4,
  effect: s.statusEffect({
    damagePerTurn: 8,
    damageType: "fire",
    duration: 3,
    color: "#ff6600"
  }),
  effectDuration: 3,
  targetType: "enemy"
});

Bombardment (Siege)

A catapult ability with long range, massive AOE, and a gold cost per use.

s.ability("bombardment", {
  damage: 80,
  damageType: "physical",
  range: 6,
  aoeRadius: 2,
  cooldown: 4,
  cost: { gold: 30 },
  targetType: "any"
});

Cyberpunk Scenario Examples

The Cyberpunk scenario features tech-based abilities with energy costs and status effects.

EMP (Disable AOE)

Disables all mechanical units in an area, blocking their movement and attacks.

s.ability("emp", {
  damage: 10,
  damageType: "energy",
  range: 4,
  aoeRadius: 3,
  cooldown: 4,
  cost: { energy: 40 },
  effect: s.statusEffect({
    duration: 2,
    blocksMove: true,
    blocksAttack: true,
    color: "#00bfff"
  }),
  effectDuration: 2,
  targetType: "enemy"
});

Overclock (Self-Buff)

A cyborg boosts its own stats temporarily. Targets self (ally) with a beneficial status.

s.ability("overclock", {
  range: 0,
  cooldown: 3,
  cost: { energy: 20 },
  effect: s.statusEffect({
    duration: 2,
    color: "#ff00ff"
  }),
  effectDuration: 2,
  targetType: "ally"
});
// Note: the stat bonuses are applied through the effect's
// integration with the unit's base stats (+attack, +speed).

Data Steal (Building Disable)

A hacker ability that targets enemy buildings, disabling them temporarily.

s.ability("dataSteal", {
  damage: 0,
  range: 3,
  cooldown: 5,
  cost: { data: 25 },
  effect: s.statusEffect({
    duration: 3,
    blocksAttack: true,
    color: "#9900ff"
  }),
  effectDuration: 3,
  targetType: "building"
});

Plasma Beam (Single Target)

A focused high-damage energy beam with a damage multiplier.

s.ability("plasmaBeam", {
  damage: 50,
  damageMulti: 1.5,
  damageType: "energy",
  range: 5,
  cooldown: 3,
  cost: { energy: 30 },
  targetType: "enemy"
});

Self-Destruct (Suicide AOE)

A drone detonates itself, dealing massive damage in a wide area. The unit is destroyed after use.

s.ability("selfDestruct", {
  damage: 100,
  damageType: "energy",
  range: 0,
  aoeRadius: 3,
  cooldown: 1,
  targetType: "any"
});
// The unit's hp is set to 0 after activation via an event hook.
// See Events for on-ability-used triggers.

Self-destruct abilities typically use range: 0 (centered on self) and targetType: "any" so they hit everything nearby. Wire the unit's death to an event that fires after the ability resolves.
