Biomes

How biomes shape gameplay — terrain types control movement, combat, vision, hazards, and resource placement across the map.

How Biomes Affect Gameplay

Every hex on the map belongs to a biome. Biomes are far more than visual paint — they are the primary lever for shaping strategic decisions. A well-designed biome set forces players to think about where they move, not just what they build.

  • Movement — moveCost determines how many movement points a unit spends entering a hex. Higher values slow units down, creating natural chokepoints and safe corridors.
  • Combat bonuses — defenseBonus and attackBonus modify a unit's stats while occupying the hex. Forests might grant defense; high ground might grant attack.
  • Vision — visionBonus extends a unit's sight range (hilltops, watchtowers), while visionPenalty reduces it (dense jungle, fog).
  • Damage — damagePerTurn inflicts automatic damage on any unit that ends its turn on the hex. Use this for lava, toxic waste, radiation zones, and other environmental hazards.
  • Resource placement — harvestBonus multiplies the yield when a unit or building harvests resources on that hex. Fertile plains produce more; barren rock produces less.
  • Access control — impassable blocks all units entirely, and allowedUnits restricts passage to specific unit types (e.g., only flying units can cross mountains, only ships can cross water).

Full Parameter Reference

Pass these options as the second argument to s.biome().

Parameter Type Default Description
color string — Hex color used to render the biome on the map. Required.
moveCost number 1 Movement points spent to enter this hex. Higher values slow units down.
defenseBonus number 0 Flat bonus added to a unit's defense while on this hex.
attackBonus number 0 Flat bonus added to a unit's attack while on this hex.
harvestBonus number 1.0 Multiplier applied to resource harvest yields on this hex.
visionBonus number 0 Extra vision range granted to units on this hex.
visionPenalty number 0 Vision range reduction for units on this hex (subtracted from base vision).
damagePerTurn number 0 HP lost by any unit that ends its turn on this hex.
impassable boolean false If true, no unit can enter this hex (unless overridden by allowedUnits).
allowedUnits array [] List of unit IDs that can enter an otherwise impassable hex. Ignored if impassable is false.
spawnWeight number 1.0 Relative likelihood that resources and neutral units spawn on this biome.

Balance Tips

Keep moveCost between 1 and 3.

A moveCost of 1 is open terrain — fast and easy to cross. A cost of 2 meaningfully slows units. A cost of 3 makes the hex feel like a slog. Going above 3 is rarely fun unless you want near-impassable terrain without fully blocking it.

Defense bonuses of 0–30 cover most cases.

A bonus of 10 is a gentle nudge ("forests are slightly safer"). A bonus of 20–30 creates strong defensive positions that players will fight over. Going above 30 can make entrenched units nearly unkillable without siege or abilities.

Use damagePerTurn sparingly for hazard biomes.

5–10 damage per turn creates a soft hazard that punishes lingering. 15+ makes the hex genuinely dangerous. Pair with allowedUnits to let specific unit types (dragons, hazmat troopers) ignore the penalty.

Castle Scenario Examples

The built-in Castle scenario defines five biomes that create a classic fantasy battlefield.

Plains

The basic, unmodified biome. Fast movement, no bonuses, no penalties — the default "neutral ground."

s.biome("plains", {
  color: "#7ec850",
  moveCost: 1
});

Forest

A defensive biome. Units in forests are harder to kill but move more slowly. Ideal for ambushes and holding positions.

s.biome("forest", {
  color: "#2d6a1e",
  moveCost: 2,
  defenseBonus: 20,
  visionPenalty: 1
});

Mountain

A strategic high-ground biome. Grants vision and attack bonuses but costs heavily to enter. Most units cannot cross; only specific types are allowed.

s.biome("mountain", {
  color: "#8a8a8a",
  moveCost: 3,
  defenseBonus: 10,
  attackBonus: 5,
  visionBonus: 2,
  impassable: true,
  allowedUnits: ["dragon"]
});

Water

Fully impassable. No unit can enter, creating hard boundaries on the map that force flanking and bridge control.

s.biome("water", {
  color: "#2389da",
  impassable: true
});

Lava

A hazardous biome that damages any unit on it — except dragons, which are immune. Use lava to guard high-value areas or as a natural moat.

s.biome("lava", {
  color: "#d4440f",
  moveCost: 2,
  damagePerTurn: 15,
  allowedUnits: ["dragon"]
});
Pair hazards with allowedUnits.

Lava that only dragons can survive gives the dragon unit a unique strategic role — scouting or attacking across terrain that is deadly to everyone else.

Cyberpunk Scenario Examples

The Cyberpunk scenario re-skins biome mechanics for a post-apocalyptic urban setting.

Wasteland

Open ground with minimal cover. The Cyberpunk equivalent of plains.

s.biome("wasteland", {
  color: "#8c7a5e",
  moveCost: 1,
  harvestBonus: 0.5
});

Ruins

Crumbling buildings provide cover and slow movement. A defensive biome similar to forests.

s.biome("ruins", {
  color: "#5a5a6e",
  moveCost: 2,
  defenseBonus: 15,
  visionPenalty: 1
});

Toxic Zone

Irradiated areas that damage units each turn. Only hazmat-equipped units (mutants) can survive here long-term.

s.biome("toxic_zone", {
  color: "#4ade40",
  moveCost: 2,
  damagePerTurn: 10,
  allowedUnits: ["mutant"],
  spawnWeight: 0.5
});

Data Stream

A network corridor that boosts vision and attack for hacker units. Fast to traverse, but offers no physical defense.

s.biome("data_stream", {
  color: "#00e5ff",
  moveCost: 1,
  attackBonus: 10,
  visionBonus: 2
});

Bunker

Hardened defensive positions. Slow to enter but extremely strong once occupied.

s.biome("bunker", {
  color: "#3d3d4d",
  moveCost: 3,
  defenseBonus: 30,
  attackBonus: 5
});

Biomes in Map Generation

When configuring s.mapgen(), you assign a weight to each biome. The map generator uses these weights to decide how frequently each biome appears. Weights are relative proportions and must sum to 1.0.

s.mapgen({
  size: 48,
  biomes: {
    plains:   { weight: 0.40 },
    forest:   { weight: 0.25 },
    mountain: { weight: 0.15 },
    water:    { weight: 0.10 },
    lava:     { weight: 0.10 }
  }
});
Weights must sum to 1.0.

If your weights add up to more or less than 1.0, the map generator will normalize them, but the results may not match your intent. Always double-check the arithmetic.

A few guidelines for weight distribution:

  • Give your primary biome (the one players move through most) the highest weight — typically 0.30–0.50.
  • Keep hazard and impassable biomes under 0.15 each. Too many blocking hexes makes the map frustrating.
  • Use defensive biomes (forests, ruins, bunkers) at 0.15–0.25 to create strategic variety without overwhelming the map.
  • Test your distribution at multiple map sizes. Weights that work at size: 32 may create too many isolated pockets at size: 64.
